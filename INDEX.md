## 🔄 Workflow файлы

### 7. `vk-post-workflow.json`
**Базовый workflow - один пост, одно изображение**

```
Сложность: ⭐ Начальный
Узлов: 5
Время настройки: 5 минут
```

**Когда использовать:**
- Простая публикация постов
- Обучение работе с n8n
- Тестирование интеграции

**Структура:**
```
Manual Trigger → Set Data → Download Image → Upload to VK → Create Post
```

---

### 8. `vk-post-multiple-images-workflow.json`
**Продвинутый workflow - несколько изображений**

```
Сложность: ⭐⭐ Средний
Узлов: 9
Время настройки: 10 минут
```

**Когда использовать:**
- Публикация фотоотчетов
- Галереи изображений
- Коллекции фото (до 10 шт)

**Структура:**
```
Manual Trigger → Set Data → Prepare → Split → Download → 
Upload → Aggregate → Format → Create Post
```

---

### 9. `vk-scheduled-post-workflow.json`
**Автоматизированный workflow с расписанием**

```
Сложность: ⭐⭐⭐ Продвинутый
Узлов: 10
Время настройки: 15 минут
```

**Когда использовать:**
- Автоматическая публикация по расписанию
- Интеграция с CMS/API
- Контент-планирование
- Уведомления об ошибках

**Структура:**
```
Schedule → Get Content → Validate → Download → Upload → Post
                ↓                                        ↓
         Error Handler → Telegram Notification   Success → Telegram
```

## ❓ Частые вопросы (FAQ)

### Общие вопросы

**Q: Нужно ли мне знать программирование?**
A: Нет, базовые workflow работают без кода. Для продвинутых функций полезно знать JavaScript.

**Q: Сколько стоит использование?**
A: n8n бесплатен (self-hosted), VK API бесплатен. Нужен только сервер для n8n.

**Q: Можно ли публиковать в несколько групп?**
A: Да, см. [EXAMPLES.md - Мультигрупповая публикация](EXAMPLES.md)

**Q: Как часто можно публиковать?**
A: Лимит VK API: 3 запроса/сек для пользователя, 20 запросов/сек для группы.

### Технические вопросы

**Q: Какая версия n8n нужна?**
A: Минимум 0.200.0, рекомендуется последняя стабильная.

**Q: Поддерживается ли видео?**
A: Текущая версия поддерживает только изображения. Видео - в планах.

**Q: Можно ли редактировать опубликованные посты?**
A: Да, используйте метод `wall.edit` в custom HTTP Request узле.

**Q: Как удалить пост?**
A: Используйте метод `wall.delete` в custom HTTP Request узле.
