# Быстрый старт - VK Automation с n8n

Это руководство поможет вам быстро настроить автоматизацию публикации постов в VK за 15 минут.

## Шаг 1: Установка n8n (5 минут)

### Вариант A: Docker (рекомендуется)

```bash
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n
```

### Вариант B: npm

```bash
npm install n8n -g
n8n start
```

Откройте браузер: http://localhost:5678

## Шаг 2: Создание VK приложения (5 минут)

1. Перейдите на https://vk.com/apps?act=manage
2. Нажмите **"Создать приложение"**
3. Выберите **"Standalone приложение"**
4. Заполните:
   - Название: "n8n Automation"
   - Категория: "Другое"
5. Нажмите **"Создать"**
6. В настройках приложения:
   - Включите **"Open API"**
   - Redirect URI: `http://localhost:5678/rest/oauth2-credential/callback`
7. Скопируйте:
   - **Client ID** (ID приложения)
   - **Secure Key** (Защищённый ключ)

## Шаг 3: Настройка credentials в n8n (3 минуты)

1. В n8n откройте **Credentials** → **New**
2. Найдите и выберите **"VK OAuth2 API"**
3. Заполните:
   - **Client ID**: вставьте из VK
   - **Client Secret**: вставьте Secure Key из VK
   - **Scope**: `wall,photos,groups`
4. Нажмите **"Connect my account"**
5. Авторизуйтесь в VK
6. Сохраните credentials

## Шаг 4: Импорт workflow (2 минуты)

1. В n8n откройте **Workflows** → **Import from File**
2. Выберите файл `vk-automation-workflow.json`
3. Workflow импортирован!

## Шаг 5: Настройка workflow (5 минут)

### 5.1 Получите ID вашей группы VK

**Вариант 1:** Из URL группы
- URL: `https://vk.com/club123456789`
- ID группы: `123456789`

**Вариант 2:** Через API
```bash
curl "https://api.vk.com/method/groups.getById?group_id=YOUR_GROUP_SCREEN_NAME&access_token=YOUR_TOKEN&v=5.131"
```

### 5.2 Настройте источник данных

Откройте узел **"Fetch Posts from Source"** и выберите один из вариантов:

#### Вариант A: Тестовые данные (для начала)

Замените узел на **"Set"** узел с данными:

```json
[
  {
    "type": "text",
    "content": "Тестовый пост из n8n!",
    "vk_group_id": "123456789"
  }
]
```

#### Вариант B: Локальный файл

Замените на узел **"Read Binary File"**:
- File Path: `/path/to/example-posts.json`

#### Вариант C: Ваш API

Настройте HTTP Request:
- URL: `https://your-api.com/posts`
- Method: GET
- Authentication: настройте если нужно

### 5.3 Укажите ID группы

Добавьте переменную окружения или укажите в каждом посте:

**Через переменную окружения:**
```bash
export VK_GROUP_ID=123456789
```

**Или в данных поста:**
```json
{
  "vk_group_id": "123456789"
}
```

## Шаг 6: Тестовый запуск (1 минута)

1. Откройте workflow в n8n
2. Нажмите **"Execute Workflow"**
3. Проверьте выполнение каждого узла
4. Убедитесь, что пост появился в VK!

## Готово! 🎉

Теперь ваш workflow готов к автоматической публикации постов в VK.

## Быстрые настройки

### Изменить расписание

Откройте узел **"Schedule Trigger"**:

- **Каждый час:** Interval = 1 hour
- **Каждый день в 9:00:** Cron = `0 9 * * *`
- **Каждый будний день в 9:00 и 18:00:** Cron = `0 9,18 * * 1-5`

### Добавить больше постов

Отредактируйте файл `example-posts.json` или настройте свой источник данных.

### Публикация в несколько групп

Добавьте узел **"Split In Batches"** и укажите разные `vk_group_id` для каждой группы.

## Частые вопросы

### Пост не публикуется

1. Проверьте VK credentials
2. Убедитесь, что `vk_group_id` правильный
3. Проверьте права доступа в VK приложении
4. Убедитесь, что вы администратор группы

### Изображение не загружается

1. Проверьте URL изображения (должен быть доступен)
2. Проверьте формат (JPG, PNG, GIF)
3. Проверьте размер (максимум 50 МБ)

### Ошибка "Access denied"

1. Переавторизуйтесь в VK через n8n credentials
2. Проверьте scope: должен быть `wall,photos,groups`
3. Убедитесь, что у вас есть права администратора группы
