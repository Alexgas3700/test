# N8N Customer Acquisition Email Campaign

Автоматизированная система email-рассылок для привлечения новых клиентов-предпринимателей с помощью персонализированных предложений со скидками.

## 📋 Описание

Этот проект содержит готовый workflow для n8n, который автоматизирует процесс отправки предложений со скидками потенциальным клиентам-предпринимателям. Система включает:

- **Автоматическую отправку**: Еженедельные рассылки каждый понедельник в 9:00
- **Персонализацию**: Индивидуальные промокоды и персонализированные сообщения
- **Фильтрацию**: Отправка только клиентам с подтвержденным согласием на маркетинг
- **Аналитику**: Логирование всех отправок для отслеживания эффективности
- **Обработку ошибок**: Автоматическое логирование ошибок

## 🚀 Быстрый старт

### Предварительные требования

1. **n8n** установлен и запущен (версия 1.0+)
2. **SMTP сервер** для отправки email
3. **База данных** с контактами предпринимателей
4. **CRM система** (опционально) для логирования кампаний

### Установка

1. Импортируйте workflow в n8n:
   ```bash
   # Откройте n8n UI
   # Перейдите в Workflows → Import from File
   # Выберите файл n8n-customer-acquisition-workflow.json
   ```

2. Настройте учетные данные в n8n:
   - SMTP credentials для отправки email
   - HTTP Basic Auth для доступа к базе данных предпринимателей
   - HTTP Header Auth для CRM API (если используется)

3. Настройте переменные окружения:
   ```bash
   # В n8n Settings → Variables добавьте:
   ENTREPRENEURS_DB_URL=https://your-database-api.com
   SENDER_EMAIL=marketing@yourcompany.com
   LANDING_PAGE_URL=https://yourcompany.com/claim-offer
   UNSUBSCRIBE_URL=https://yourcompany.com/unsubscribe
   PRIVACY_URL=https://yourcompany.com/privacy
   CRM_API_URL=https://your-crm-api.com
   ERROR_LOG_URL=https://your-logging-service.com
   BCC_EMAIL=analytics@yourcompany.com
   ```

## 📊 Структура Workflow

### Узлы (Nodes)

1. **Schedule Trigger** - Запускает workflow каждый понедельник в 9:00
2. **Fetch Potential Entrepreneurs** - Получает список потенциальных клиентов из базы данных
3. **Filter Valid Contacts** - Фильтрует контакты по критериям:
   - Наличие email адреса
   - Тип бизнеса: entrepreneur
   - Согласие на маркетинговые рассылки
4. **Prepare Email Data** - Подготавливает персонализированные данные:
   - Генерирует уникальные промокоды
   - Рассчитывает размер скидки (20% или 25% для tech-индустрии)
   - Устанавливает срок действия предложения (14 дней)
5. **Send Discount Offer Email** - Отправляет HTML email с предложением
6. **Log Campaign Activity** - Логирует отправку в CRM
7. **Error Handler** - Обрабатывает ошибки при отправке
8. **Log Error** - Логирует ошибки для анализа
9. **Campaign Summary** - Создает сводку по кампании

### Поток данных

```
Schedule Trigger
    ↓
Fetch Potential Entrepreneurs
    ↓
Filter Valid Contacts
    ↓
Prepare Email Data
    ↓
Send Discount Offer Email ──→ (error) ──→ Error Handler ──→ Log Error
    ↓
Log Campaign Activity
    ↓
Campaign Summary
```

## 🎨 Функции

### Персонализация

- **Имя получателя**: Обращение по имени
- **Название компании**: Упоминание бизнеса клиента
- **Уникальный промокод**: Формат `NEWBIZMMDD-XXXX` (дата + ID клиента)
- **Динамическая скидка**: 25% для tech-индустрии, 20% для остальных
- **Срок действия**: Автоматически устанавливается на 14 дней

### Email шаблон

Профессиональный HTML email включает:
- Привлекательный header с градиентом
- Выделенный блок с промокодом
- Список преимуществ с иконками
- Призыв к действию (CTA button)
- Footer с ссылками на отписку и политику конфиденциальности

### Безопасность и соответствие

- ✅ Проверка согласия на маркетинг (`consent_marketing`)
- ✅ Ссылка на отписку в каждом письме
- ✅ Ссылка на политику конфиденциальности
- ✅ Логирование всех отправок

## 🔧 Настройка

### Формат данных базы предпринимателей

API должен возвращать JSON с массивом объектов:

```json
[
  {
    "id": "12345",
    "email": "entrepreneur@example.com",
    "first_name": "Иван",
    "company_name": "ИП Иванов",
    "business_type": "entrepreneur",
    "industry": "tech",
    "consent_marketing": true,
    "email_verified": true
  }
]
```

### Параметры API запроса

Workflow автоматически добавляет query параметры:
- `status=potential` - только потенциальные клиенты
- `segment=entrepreneur` - только предприниматели
- `email_verified=true` - только с подтвержденным email

### SMTP настройки

В n8n создайте SMTP credentials:
- **Host**: smtp.yourprovider.com
- **Port**: 587 (или 465 для SSL)
- **User**: your-smtp-username
- **Password**: your-smtp-password
- **Secure**: true (для SSL/TLS)

## 📈 Аналитика и отслеживание

### Логирование кампаний

Каждая отправка логируется в CRM со следующими данными:
```json
{
  "email": "entrepreneur@example.com",
  "campaign_type": "send_discount_offer",
  "discount_code": "NEWBIZ0210-5678",
  "sent_at": "2026-02-10T09:00:00.000Z",
  "status": "sent"
}
```

### Метрики

Workflow создает сводку кампании:
- Общее количество отправленных писем
- Дата и время кампании
- Тип кампании

### Отслеживание ошибок

Все ошибки логируются с информацией:
- ID workflow
- Сообщение об ошибке
- Email получателя
- Временная метка

## 🎯 Критерии целевой аудитории

Workflow отправляет письма только клиентам, которые соответствуют всем критериям:

1. ✅ **Email адрес** - не пустой
2. ✅ **Тип бизнеса** - entrepreneur
3. ✅ **Согласие на маркетинг** - true
4. ✅ **Подтвержденный email** - true (проверяется в API запросе)

## 🔄 Расписание

По умолчанию workflow запускается:
- **Частота**: Еженедельно
- **День**: Понедельник
- **Время**: 09:00
- **Часовой пояс**: Europe/Moscow

Для изменения расписания отредактируйте cron expression в узле "Schedule Trigger":
```
0 9 * * 1  # Каждый понедельник в 9:00
0 9 * * *  # Каждый день в 9:00
0 9 1 * *  # Первое число каждого месяца в 9:00
```

## 🛠️ Кастомизация

### Изменение email шаблона

Email шаблон находится в узле "Send Discount Offer Email". Вы можете изменить:
- Цветовую схему (градиент в `.header`)
- Текст сообщения
- Список преимуществ
- Стили кнопки CTA

### Изменение логики скидок

В узле "Prepare Email Data" настройте:
```javascript
// Текущая логика:
discount_amount = industry === 'tech' ? '25' : '20'

// Примеры изменений:
// По размеру компании:
discount_amount = company_size > 50 ? '30' : '20'

// По региону:
discount_amount = region === 'moscow' ? '25' : '20'

// Фиксированная скидка:
discount_amount = '20'
```

### Изменение формата промокода

```javascript
// Текущий формат: NEWBIZ0210-5678
'NEWBIZ' + $now.format('MMDD') + '-' + $json.id.toString().slice(-4).toUpperCase()

// Примеры:
// Простой: ENTREPRENEUR2026
'ENTREPRENEUR' + $now.format('YYYY')

// С названием компании: IVANOV-20
$json.company_name.toUpperCase().slice(0, 6) + '-' + discount_amount
```

## 🐛 Устранение неполадок

### Проблема: Письма не отправляются

**Решения:**
1. Проверьте SMTP credentials
2. Убедитесь, что SMTP порт открыт
3. Проверьте лимиты SMTP провайдера
4. Проверьте логи ошибок в n8n

### Проблема: Получатели не получают письма

**Решения:**
1. Проверьте spam папку
2. Убедитесь в корректности email адресов
3. Проверьте SPF/DKIM записи домена
4. Используйте email validation сервис

### Проблема: API не возвращает данные

**Решения:**
1. Проверьте URL и credentials
2. Проверьте формат query параметров
3. Убедитесь, что API доступен
4. Проверьте лимиты API

### Проблема: Промокоды не уникальны

**Решения:**
1. Добавьте больше случайности в генерацию
2. Используйте UUID вместо ID клиента
3. Проверяйте уникальность перед отправкой
