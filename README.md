# VK Post Automation with n8n

Автоматизация публикации постов в VK (ВКонтакте) с использованием n8n. Поддерживает текстовые посты и посты с изображениями.

## Возможности

- ✅ Автоматическая публикация текстовых постов в VK
- ✅ Автоматическая публикация постов с изображениями в VK
- ✅ Планирование публикаций по расписанию
- ✅ Фильтрация типов постов (text, image)
- ✅ Загрузка изображений из внешних источников
- ✅ Логирование результатов публикации
- ✅ Поддержка публикации от имени группы

## Требования

- n8n (версия 1.0.0 или выше)
- VK API токен с правами доступа:
  - `wall` - публикация на стене
  - `photos` - загрузка фотографий
  - `groups` - управление группами (если публикуете в группу)
- Источник контента (API, база данных, webhook и т.д.)

## Установка

### 1. Импорт workflow в n8n

1. Откройте n8n
2. Перейдите в раздел "Workflows"
3. Нажмите "Import from File"
4. Выберите файл `vk-automation-workflow.json`
5. Workflow будет импортирован и готов к настройке

### 2. Настройка VK OAuth2 Credentials

1. Создайте VK приложение:
   - Перейдите на https://vk.com/apps?act=manage
   - Нажмите "Создать приложение"
   - Выберите тип "Standalone приложение"
   - Укажите название и описание

2. Настройте приложение:
   - В настройках приложения включите "Open API"
   - Добавьте Redirect URI: `https://your-n8n-instance.com/rest/oauth2-credential/callback`
   - Скопируйте Client ID и Secure Key

3. Добавьте credentials в n8n:
   - Перейдите в "Credentials" → "New"
   - Выберите "VK OAuth2 API"
   - Введите Client ID и Client Secret
   - Укажите необходимые права доступа: `wall,photos,groups`
   - Нажмите "Connect my account" и авторизуйтесь

### 3. Настройка источника контента

Измените узел "Fetch Posts from Source" для получения постов из вашего источника:

**Вариант 1: REST API**
```json
{
  "url": "https://your-api.com/posts",
  "method": "GET",
  "authentication": "headerAuth"
}
```

**Вариант 2: База данных**
- Замените HTTP Request узел на узел базы данных (PostgreSQL, MySQL, MongoDB и т.д.)
- Настройте SQL запрос для получения постов

**Вариант 3: Webhook**
- Замените Schedule Trigger на Webhook узел
- Настройте webhook URL для получения постов

### 4. Формат данных

Ваш источник контента должен возвращать данные в следующем формате:

**Текстовый пост:**
```json
{
  "type": "text",
  "content": "Текст вашего поста",
  "vk_group_id": "123456789"
}
```

**Пост с изображением:**
```json
{
  "type": "image",
  "content": "Текст поста с изображением",
  "image_url": "https://example.com/image.jpg",
  "vk_group_id": "123456789"
}
```

### 5. Настройка переменных окружения

Добавьте следующие переменные окружения в n8n:

```bash
VK_GROUP_ID=123456789  # ID вашей группы VK (без знака минус)
```

Или укажите `vk_group_id` в каждом посте.

## Структура Workflow

### Узлы workflow:

1. **Schedule Trigger** - Запускает workflow по расписанию (каждый час)
2. **Fetch Posts from Source** - Получает посты из источника контента
3. **Filter Post Types** - Фильтрует только текстовые и изображения посты
4. **Check if Image Post** - Проверяет тип поста (текст или изображение)
5. **Download Image** - Загружает изображение из URL
6. **Get VK Upload Server** - Получает URL сервера для загрузки фото
7. **Upload Image to VK** - Загружает изображение на сервер VK
8. **Save Wall Photo** - Сохраняет фото для публикации на стене
9. **Format Attachment** - Форматирует attachment для VK API
10. **Merge Post Data** - Объединяет данные поста с attachment
11. **Post to VK (Text Only)** - Публикует текстовый пост
12. **Post to VK (Final)** - Публикует пост с изображением
13. **Merge All Results** - Объединяет результаты всех публикаций
14. **Log Results** - Логирует результаты публикации

### Логика работы:

```
Расписание → Получение постов → Фильтрация → Проверка типа
                                                    ↓
                                    ┌───────────────┴───────────────┐
                                    ↓                               ↓
                              Текстовый пост                  Изображение
                                    ↓                               ↓
                            Публикация в VK          Загрузка → Публикация в VK
                                    ↓                               ↓
                                    └───────────────┬───────────────┘
                                                    ↓
                                            Логирование результатов
```

## Настройка расписания

По умолчанию workflow запускается каждый час. Чтобы изменить расписание:

1. Откройте узел "Schedule Trigger"
2. Измените параметры:
   - **Каждый день в определенное время:** выберите "Every Day" и укажите время
   - **Каждую неделю:** выберите "Every Week" и укажите дни недели
   - **Cron выражение:** выберите "Cron" для более гибкой настройки

Примеры cron выражений:
- `0 9 * * *` - каждый день в 9:00
- `0 9,18 * * *` - каждый день в 9:00 и 18:00
- `0 9 * * 1-5` - каждый будний день в 9:00

## Расширенные возможности

### Публикация в несколько групп

Добавьте узел "Split In Batches" после получения постов:

```javascript
// В узле Code
const groups = ['123456789', '987654321'];
const results = [];

for (const group of groups) {
  results.push({
    json: {
      ...items[0].json,
      vk_group_id: group
    }
  });
}

return results;
```

### Добавление хештегов

Модифицируйте текст поста перед публикацией:

```javascript
// В узле Code перед публикацией
const hashtags = ['#автоматизация', '#n8n', '#vk'];
const content = items[0].json.content;
const contentWithHashtags = `${content}\n\n${hashtags.join(' ')}`;

return [{
  json: {
    ...items[0].json,
    content: contentWithHashtags
  }
}];
```

### Отложенная публикация

VK API поддерживает отложенную публикацию. Добавьте параметр `publish_date`:

```json
{
  "additionalFields": {
    "from_group": true,
    "signed": false,
    "publish_date": 1234567890
  }
}
```

Где `publish_date` - Unix timestamp времени публикации.

### Обработка ошибок

Добавьте узел "Error Trigger" для обработки ошибок:

1. Добавьте узел "Error Trigger"
2. Подключите его к узлу отправки уведомлений (Email, Telegram, Slack и т.д.)
3. Настройте шаблон уведомления об ошибке

## Устранение неполадок

### Ошибка "Invalid access token"

- Проверьте, что VK OAuth2 credentials настроены правильно
- Убедитесь, что токен не истек
- Переавторизуйтесь в VK через n8n credentials

### Ошибка "Access denied"

- Проверьте права доступа приложения VK
- Убедитесь, что у вас есть права на публикацию в группе
- Проверьте, что `vk_group_id` указан правильно

### Изображение не загружается

- Проверьте, что URL изображения доступен
- Убедитесь, что формат изображения поддерживается (JPG, PNG, GIF)
- Проверьте размер изображения (максимум 50 МБ)

### Пост не публикуется

- Проверьте формат данных из источника контента
- Убедитесь, что поле `type` имеет значение "text" или "image"
- Проверьте, что поле `content` не пустое

## Безопасность

### Рекомендации:

1. **Не храните токены в открытом виде** - используйте n8n credentials
2. **Ограничьте права доступа** - запрашивайте только необходимые права
3. **Используйте HTTPS** - для всех API запросов
4. **Регулярно обновляйте токены** - следите за сроком действия
5. **Логируйте действия** - для аудита и отладки
