# n8n Email Campaign Workflow

Автоматизированный workflow для проведения email-рассылок с использованием n8n.

## 📋 Описание

Этот workflow предназначен для автоматизации email-рассылок с следующими возможностями:

- ✉️ Персонализированные email-рассылки
- 📊 Интеграция с Google Sheets для управления контактами
- 🔄 Автоматическая обработка партиями для предотвращения блокировки
- 📈 Логирование результатов и статистика
- 🎯 Фильтрация активных контактов
- 📧 HTML-шаблоны писем с адаптивным дизайном
- 📊 Автоматические отчеты о результатах кампании

## 🚀 Установка и настройка

### Шаг 1: Импорт workflow в n8n

1. Откройте n8n
2. Нажмите на "Import from File" или "Import from URL"
3. Выберите файл `email-campaign-workflow.json`
4. Workflow будет импортирован в вашу n8n инстанцию

### Шаг 2: Настройка учетных данных

#### Google Sheets API

1. Создайте проект в [Google Cloud Console](https://console.cloud.google.com/)
2. Включите Google Sheets API
3. Создайте OAuth 2.0 credentials
4. В n8n добавьте credentials для Google Sheets:
   - Перейдите в Settings → Credentials
   - Нажмите "New Credential"
   - Выберите "Google Sheets OAuth2 API"
   - Введите Client ID и Client Secret
   - Авторизуйтесь через Google

#### SMTP для отправки email

1. В n8n добавьте SMTP credentials:
   - Перейдите в Settings → Credentials
   - Нажмите "New Credential"
   - Выберите "SMTP"
   - Введите данные вашего SMTP сервера:
     - Host (например, smtp.gmail.com)
     - Port (обычно 587 для TLS)
     - User (ваш email)
     - Password (пароль приложения)

**Примеры SMTP серверов:**
- Gmail: smtp.gmail.com:587
- Outlook: smtp-mail.outlook.com:587
- SendGrid: smtp.sendgrid.net:587
- Mailgun: smtp.mailgun.org:587

### Шаг 3: Подготовка Google Sheets

Создайте Google Sheets таблицу со следующими колонками:

| A (Email) | B (Name) | C (Company) | D (Status) | E (Last Contact) | F (Last Email Sent) | G (Email Status) | H (Error Timestamp) | I (Error Message) |
|-----------|----------|-------------|------------|------------------|---------------------|------------------|---------------------|-------------------|
| john@example.com | John Doe | Acme Corp | Active | 2026-01-15 | | | | |
| jane@example.com | Jane Smith | Tech Inc | Active | 2026-01-20 | | | | |
| bob@example.com | Bob Johnson | StartUp LLC | Inactive | 2025-12-10 | | | | |

**Важно:**
- Колонка D (Status) должна содержать "Active" для контактов, которым нужно отправить письма
- Колонки F-I будут автоматически заполняться workflow

## 🔧 Настройка workflow

### 1. Настройка узла "Get Email List"

- Укажите ID вашей Google Sheets таблицы
- Убедитесь, что диапазон установлен правильно (A:E)

### 2. Настройка узла "Send Email"

Измените параметры отправителя:
```json
{
  "fromEmail": "your-email@yourcompany.com"
}
```

### 3. Персонализация шаблона письма

В узле "Personalize Emails" можно изменить:
- Тему письма
- HTML-шаблон
- Стили оформления
- Ссылки и кнопки

### 4. Настройка батчей

В узле "Split In Batches" можно изменить:
- `batchSize`: количество писем в одной партии (по умолчанию 10)

В узле "Wait Between Batches" можно изменить:
- Время ожидания между партиями (по умолчанию 5 секунд)

### 5. Настройка отчетов

В узле "Send Report" укажите email администратора:
```json
{
  "toEmail": "admin@yourcompany.com"
}
```

## 📊 Структура workflow

```
1. Manual Trigger → Запуск workflow вручную
2. Get Email List → Получение списка контактов из Google Sheets
3. Filter Active Contacts → Фильтрация только активных контактов
4. Personalize Emails → Создание персонализированных писем
5. Split In Batches → Разделение на партии
6. Wait Between Batches → Пауза между партиями
7. Send Email → Отправка письма
8. Log Success/Error → Логирование результата
9. Update Google Sheet → Обновление таблицы
10. Campaign Summary → Создание сводки
11. Send Report → Отправка отчета администратору
```

## 🎯 Использование

### Запуск вручную

1. Откройте workflow в n8n
2. Нажмите "Execute Workflow"
3. Workflow обработает все активные контакты из Google Sheets

### Автоматический запуск (расписание)

Чтобы настроить автоматический запуск по расписанию:

1. Замените узел "Manual Trigger" на "Cron" или "Schedule Trigger"
2. Настройте расписание (например, каждый понедельник в 9:00)
3. Активируйте workflow

Пример настройки Cron:
```
0 9 * * 1  # Каждый понедельник в 9:00
```

## 📧 Персонализация писем

Workflow поддерживает следующие переменные персонализации:

- `{{ $json.Name }}` - Имя получателя
- `{{ $json.Company }}` - Название компании
- `{{ $json.Email }}` - Email получателя
- `{{ $json.LastContact }}` - Дата последнего контакта

Пример использования в шаблоне:

```html
<p>Здравствуйте, {{ $json.Name }}!</p>
<p>Мы рады предложить специальные условия для {{ $json.Company }}.</p>
```

## 📈 Отчеты и аналитика

После завершения кампании вы получите:

1. **Email-отчет** с информацией:
   - Дата кампании
   - Общее количество обработанных контактов
   - Количество успешно отправленных писем
   - Количество ошибок
   - Процент успешной доставки

2. **Обновленная Google Sheets** с данными:
   - Дата последней отправки
   - Статус отправки
   - Информация об ошибках (если были)

## 🔒 Безопасность и лучшие практики

### Предотвращение блокировки

- Workflow автоматически разделяет рассылку на партии
- Между партиями есть паузы (5 секунд)
- Рекомендуется не отправлять более 100-200 писем в час

### Compliance с законами

- Всегда включайте ссылку для отписки
- Убедитесь, что у вас есть согласие получателей
- Соблюдайте GDPR, CAN-SPAM и другие законы

### Тестирование

Перед массовой рассылкой:

1. Создайте тестовую таблицу с 2-3 контактами
2. Используйте свои email-адреса для тестирования
3. Проверьте, что письма приходят корректно
4. Проверьте отображение на разных устройствах

## 🛠️ Расширенные возможности

### Добавление A/B тестирования

Можно добавить узел для случайного выбора одного из двух вариантов письма:

```javascript
// В узле Code
const variants = ['A', 'B'];
const variant = variants[Math.floor(Math.random() * variants.length)];

return {
  json: {
    ...item.json,
    variant: variant,
    subject: variant === 'A' ? 'Вариант A' : 'Вариант B'
  }
};
```

### Интеграция с CRM

- Salesforce
- HubSpot
- Pipedrive
- Zoho CRM

### Отслеживание открытий и кликов

Для отслеживания можно:
1. Добавить tracking pixel в HTML письма
2. Использовать UTM-метки в ссылках
3. Интегрировать с сервисами аналитики

## 🐛 Устранение неполадок

### Письма не отправляются

1. Проверьте SMTP credentials
2. Убедитесь, что порт не заблокирован файрволом
3. Проверьте лимиты вашего SMTP провайдера

### Ошибки при чтении Google Sheets

1. Проверьте права доступа к таблице
2. Убедитесь, что Google Sheets API включен
3. Проверьте правильность ID таблицы

### Письма попадают в спам

1. Настройте SPF, DKIM, DMARC записи для вашего домена
2. Используйте профессиональный SMTP сервис
3. Избегайте спам-слов в теме и тексте
4. Убедитесь, что у вас есть согласие получателей
