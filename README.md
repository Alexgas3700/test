# VK Post Workflow для n8n

Этот workflow позволяет автоматически публиковать посты с текстом и изображениями в VK (ВКонтакте) через n8n.

## Возможности

- ✅ Публикация текстовых постов
- ✅ Загрузка и прикрепление изображений к постам
- ✅ Поддержка публикации в группы VK
- ✅ OAuth2 аутентификация

## Структура workflow

Workflow состоит из 5 узлов:

1. **Manual Trigger** - Запуск workflow вручную для тестирования
2. **Set Post Data** - Настройка данных поста (текст, URL изображения, ID группы)
3. **Download Image** - Загрузка изображения по URL
4. **Upload Photo to VK** - Загрузка фото на сервера VK
5. **Create VK Post** - Создание поста с текстом и прикрепленным изображением

## Предварительные требования

1. **n8n** установлен и запущен
2. **VK приложение** создано в VK Developers
3. **OAuth2 токен** для доступа к VK API

## Настройка VK приложения

### Шаг 1: Создание приложения VK

1. Перейдите на [VK Developers](https://dev.vk.com/)
2. Нажмите "Создать приложение"
3. Выберите тип "Standalone приложение"
4. Заполните название и описание
5. После создания сохраните:
   - **App ID** (ID приложения)
   - **Secure Key** (Защищенный ключ)

### Шаг 2: Настройка прав доступа

В настройках приложения включите следующие права:
- `photos` - для загрузки фотографий
- `wall` - для публикации постов
- `groups` - для работы с группами (если публикуете в группу)

### Шаг 3: Получение токена доступа

#### Вариант 1: Через OAuth2 в n8n (рекомендуется)

1. В n8n перейдите в **Credentials**
2. Создайте новый credential типа **VK OAuth2 API**
3. Заполните:
   - **Client ID**: App ID из VK приложения
   - **Client Secret**: Secure Key из VK приложения
   - **Scope**: `photos,wall,groups,offline`
4. Нажмите "Connect my account" и авторизуйтесь

#### Вариант 2: Получение токена вручную

Используйте Implicit Flow для получения токена:

```
https://oauth.vk.com/authorize?client_id=YOUR_APP_ID&display=page&redirect_uri=https://oauth.vk.com/blank.html&scope=photos,wall,groups,offline&response_type=token&v=5.131
```

Замените `YOUR_APP_ID` на ваш App ID и перейдите по ссылке. После авторизации токен будет в URL.

## Установка workflow

### Способ 1: Импорт через UI

1. Откройте n8n
2. Нажмите на меню (три точки) → **Import from File**
3. Выберите файл `vk-post-workflow.json`
4. Workflow будет импортирован

### Способ 2: Импорт через URL

1. В n8n нажмите **Import from URL**
2. Вставьте URL к файлу `vk-post-workflow.json`
3. Нажмите **Import**

## Настройка workflow

### 1. Настройка credentials

В узлах **Upload Photo to VK** и **Create VK Post**:
1. Кликните на узел
2. В поле **Credential to connect with** выберите созданный VK OAuth2 credential
3. Сохраните изменения

### 2. Настройка данных поста

В узле **Set Post Data** измените значения:

```json
{
  "postText": "Ваш текст поста",
  "imageUrl": "https://example.com/your-image.jpg",
  "groupId": "YOUR_GROUP_ID"
}
```

**Как получить Group ID:**
- Откройте вашу группу VK
- ID группы указан в URL: `vk.com/club123456789` (используйте `123456789`)
- Или в настройках группы → Адрес страницы

### 3. Параметры узлов

#### Upload Photo to VK
- **Resource**: Photo
- **Operation**: Upload
- **Group ID**: `={{ $('Set Post Data').item.json.groupId }}`
- **Binary Property**: `data`

#### Create VK Post
- **Resource**: Post
- **Operation**: Create
- **Group ID**: `={{ $('Set Post Data').item.json.groupId }}`
- **Message**: `={{ $('Set Post Data').item.json.postText }}`
- **Attachments**: `={{ $json.id }}`

## Использование

### Базовое использование

1. Откройте workflow в n8n
2. Нажмите **Execute Workflow** (или кнопку "Test workflow")
3. Workflow выполнится и создаст пост в VK

### Публикация с изображением

Workflow автоматически:
1. Загружает изображение по указанному URL
2. Загружает его на сервера VK
3. Прикрепляет к посту
4. Публикует пост с текстом и изображением

### Публикация только текста

Если нужно опубликовать только текст без изображения:
1. Удалите узлы **Download Image** и **Upload Photo to VK**
2. В узле **Create VK Post** удалите поле **Attachments**
3. Соедините **Set Post Data** напрямую с **Create VK Post**

## Расширенные возможности

### Публикация нескольких изображений

Для публикации нескольких изображений:

1. Добавьте узел **Split In Batches** после **Set Post Data**
2. Передайте массив URL изображений
3. Используйте **Loop Over Items** для загрузки каждого изображения
4. Соберите все ID загруженных фото
5. Передайте их в поле **Attachments** через запятую: `photo123_456,photo789_012`

### Автоматическая публикация по расписанию

Замените **Manual Trigger** на:
- **Cron** - для публикации по расписанию
- **Webhook** - для публикации через HTTP запрос
- **RSS Feed** - для автоматической публикации из RSS

Пример с Cron:
```
0 9 * * * - каждый день в 9:00
0 */4 * * * - каждые 4 часа
```

### Интеграция с другими сервисами

Вы можете добавить узлы для:
- **Google Sheets** - чтение данных постов из таблицы
- **Airtable** - управление контент-планом
- **Telegram** - получение уведомлений о публикации
- **OpenAI** - генерация текста постов
- **Unsplash/Pexels** - автоматический подбор изображений

## Обработка ошибок

### Ошибка аутентификации

```
Error: Invalid access token
```

**Решение:**
1. Проверьте, что токен действителен
2. Убедитесь, что у токена есть необходимые права (`photos`, `wall`, `groups`)
3. Пересоздайте credential в n8n

### Ошибка загрузки изображения

```
Error: Failed to download image
```

**Решение:**
1. Проверьте, что URL изображения доступен
2. Убедитесь, что изображение в поддерживаемом формате (JPG, PNG, GIF)
3. Проверьте размер изображения (макс. 50 МБ для VK)

### Ошибка публикации

```
Error: Access denied
```

**Решение:**
1. Проверьте, что у вас есть права администратора группы
2. Убедитесь, что Group ID указан правильно
3. Проверьте настройки приложения VK

## Ограничения VK API

- Максимальный размер изображения: 50 МБ
- Поддерживаемые форматы: JPG, PNG, GIF
- Максимальная длина текста поста: 16384 символа
- Максимум 10 вложений в одном посте

## Безопасность

⚠️ **Важно:**
- Никогда не публикуйте токены доступа в открытом виде
- Используйте environment variables для хранения чувствительных данных
- Регулярно обновляйте токены доступа
- Ограничьте права доступа приложения только необходимыми
