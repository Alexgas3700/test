# Telegram Channel Management Workflow для n8n

Автоматизированный workflow для управления Telegram каналом с использованием n8n.

## 📋 Описание

Этот workflow предоставляет полноценное решение для автоматизации ведения Telegram канала, включая:

- 📰 **Автоматическую публикацию контента** из RSS-лент
- ⏰ **Планирование постов** по расписанию
- 🖼️ **Поддержку различных типов контента** (текст, фото, видео)
- 🔔 **Уведомления об ошибках** для администраторов
- 🌐 **Webhook интеграцию** для внешних систем
- 🛡️ **Защиту от спама** с задержками между постами

## 🚀 Возможности

### 1. Автоматическая публикация из RSS
- Получение новостей из RSS-лент
- Фильтрация по времени публикации (только свежие новости)
- Форматирование сообщений с Markdown
- Автоматическая публикация в канал

### 2. Планировщик постов
- Запуск по расписанию (по умолчанию: 9:00, 12:00, 18:00)
- Настраиваемое расписание через cron-выражения
- Ежедневные приветственные сообщения

### 3. Webhook интеграция
- Прием сообщений через HTTP webhook
- Поддержка текстовых и медиа-сообщений
- Возможность отложенной публикации

### 4. Обработка ошибок
- Автоматические уведомления администратору при ошибках
- Проверка статуса получения данных
- Логирование проблем

## 📦 Установка

### Предварительные требования

1. **n8n** установлен и запущен
   ```bash
   npm install -g n8n
   # или
   docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n
   ```

2. **Telegram Bot Token**
   - Создайте бота через [@BotFather](https://t.me/BotFather)
   - Получите API токен
   - Добавьте бота в ваш канал как администратора

3. **ID Telegram канала**
   - Формат: `@channel_name` или числовой ID
   - Для приватных каналов используйте числовой ID (например: `-1001234567890`)

### Шаги установки

1. **Импорт workflow в n8n**
   - Откройте n8n интерфейс
   - Нажмите "Import from File"
   - Выберите файл `telegram-channel-workflow.json`

2. **Настройка Telegram API credentials**
   - Перейдите в Settings → Credentials
   - Создайте новые credentials типа "Telegram API"
   - Введите Bot Token, полученный от BotFather
   - Сохраните credentials с именем "Telegram Bot API"

3. **Настройка RSS2JSON API (опционально)**
   - Зарегистрируйтесь на [RSS2JSON](https://rss2json.com/)
   - Получите API ключ
   - Добавьте credentials в n8n

4. **Конфигурация workflow**
   - Откройте узел "Configuration"
   - Замените `@your_channel_name` на ID вашего канала
   - Укажите URL вашей RSS-ленты
   - Сохраните изменения

5. **Активация workflow**
   - Нажмите кнопку "Active" в правом верхнем углу
   - Workflow начнет работать по расписанию

## ⚙️ Конфигурация

### Основные настройки

Отредактируйте узел **Configuration** для настройки основных параметров:

```javascript
{
  "feedUrl": "https://example.com/rss",  // URL RSS-ленты
  "channelId": "@your_channel_name",     // ID вашего канала
  "adminChatId": "123456789"             // ID администратора для уведомлений
}
```

### Расписание публикаций

Измените узел **Schedule Trigger** для настройки расписания:

```
Cron выражение: 0 9,12,18 * * *
```

Примеры:
- `0 9,12,18 * * *` - 3 раза в день (9:00, 12:00, 18:00)
- `0 */2 * * *` - каждые 2 часа
- `0 10 * * 1-5` - каждый будний день в 10:00

### Форматирование сообщений

Узел **Format Message** определяет шаблон сообщений:

```markdown
## 📰 {{ $json.title }}

{{ $json.description }}

🔗 [Читать полностью]({{ $json.link }})

#новости #telegram
```

Вы можете изменить:
- Эмодзи и форматирование
- Структуру сообщения
- Хэштеги

## 🔧 Использование

### Ручной запуск

1. Откройте workflow в n8n
2. Нажмите "Execute Workflow"
3. Проверьте результаты выполнения

### Автоматический режим

После активации workflow будет:
- Запускаться по расписанию
- Получать новости из RSS
- Публиковать в канал автоматически

### Webhook интеграция

Отправьте POST запрос на webhook URL:

```bash
curl -X POST https://your-n8n-instance.com/webhook/telegram-webhook \
  -H "Content-Type: application/json" \
  -d '{
    "channelId": "@your_channel",
    "text": "Привет из webhook!",
    "type": "text"
  }'
```

Для отправки фото:

```bash
curl -X POST https://your-n8n-instance.com/webhook/telegram-webhook \
  -H "Content-Type: application/json" \
  -d '{
    "channelId": "@your_channel",
    "message": "Описание фото",
    "type": "photo",
    "media": "https://example.com/image.jpg"
  }'
```

## 📊 Структура Workflow

### Триггеры
1. **Manual Trigger** - ручной запуск для тестирования
2. **Schedule Trigger** - автоматический запуск по расписанию
3. **Webhook Trigger** - прием данных через HTTP

### Обработка данных
1. **Configuration** - основные настройки
2. **Get RSS Feed** - получение контента из RSS
3. **Process Articles** - фильтрация и обработка статей
4. **Format Message** - форматирование для Telegram

### Публикация
1. **Send to Telegram** - отправка текстовых сообщений
2. **Send Photo** - отправка фотографий
3. **Send Daily Message** - ежедневные сообщения

### Обработка ошибок
1. **Check Feed Status** - проверка успешности получения данных
2. **Error Notification** - уведомления об ошибках

## 🐛 Устранение неполадок

### Сообщения не публикуются

1. Проверьте, что бот добавлен в канал как администратор
2. Убедитесь, что Bot Token корректный
3. Проверьте ID канала (должен начинаться с @ или быть числовым)

### Ошибки при получении RSS

1. Проверьте доступность RSS-ленты
2. Убедитесь, что API ключ RSS2JSON валиден
3. Проверьте формат URL

### Webhook не работает

1. Убедитесь, что n8n доступен извне
2. Проверьте URL webhook
3. Проверьте формат отправляемых данных
