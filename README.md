# Email Marketing Workflow для n8n

Автоматизированная система email-рассылок для привлечения клиентов с использованием n8n.

## 📋 Описание

Этот workflow предназначен для автоматизации email-маркетинга и включает:

- **Сегментацию аудитории** - автоматическое разделение подписчиков на группы
- **Персонализированные шаблоны** - уникальный контент для каждого сегмента
- **Автоматическую отправку** - запуск по расписанию
- **Аналитику и отчетность** - отслеживание эффективности кампаний

## 🎯 Типы рассылок

### 1. Welcome Email (Приветственные письма)
- Отправляются новым подписчикам (до 7 дней с момента регистрации)
- Включают специальное предложение (20% скидка)
- Цель: конвертация новых лидов в клиентов

### 2. Engagement Email (Письма вовлечения)
- Для активных пользователей (взаимодействовали в последние 30 дней)
- Содержат полезный контент, новости, обновления
- Цель: поддержание интереса и лояльности

### 3. Re-engagement Email (Реактивация)
- Для неактивных подписчиков (более 30 дней без активности)
- Специальное предложение для возвращения (30% скидка)
- Цель: вернуть потерянных клиентов

## 🚀 Установка и настройка

### Шаг 1: Импорт workflow в n8n

1. Откройте n8n
2. Нажмите "Import from File"
3. Выберите файл `email-marketing-workflow.json`
4. Workflow будет добавлен в ваше рабочее пространство

### Шаг 2: Настройка учетных данных

#### SendGrid (рекомендуется)

SendGrid выбран как основная платформа для отправки email благодаря:
- Высокой доставляемости писем
- Бесплатному плану (100 писем/день)
- Подробной аналитике
- Простой интеграции с n8n

**Настройка:**

1. Зарегистрируйтесь на [SendGrid](https://sendgrid.com/)
2. Создайте API ключ:
   - Settings → API Keys → Create API Key
   - Выберите "Full Access"
   - Сохраните ключ
3. В n8n:
   - Credentials → Add Credential → SendGrid API
   - Вставьте API ключ
   - Сохраните как "SendGrid API"

#### Альтернативные платформы

Workflow также поддерживает отправку через SMTP (нода отключена по умолчанию):
- **Gmail** - для небольших объемов
- **Mailgun** - для разработчиков
- **Amazon SES** - для масштабирования
- **Postmark** - для транзакционных писем

### Шаг 3: Настройка переменных окружения

Создайте файл `.env` или настройте переменные в n8n:

```bash
# Обязательные
SENDGRID_FROM_EMAIL=your-email@domain.com
WEBSITE_URL=https://your-website.com
DOMAIN=your-domain.com

# Опциональные
AIRTABLE_BASE_ID=your-airtable-base-id
SLACK_WEBHOOK_URL=your-slack-webhook-url
WEBHOOK_URL=your-analytics-webhook-url
```

### Шаг 4: Настройка источника подписчиков

Измените ноду "Get Subscriber List" для подключения к вашему источнику данных:

**Варианты:**
- REST API вашей CRM
- База данных (PostgreSQL, MySQL)
- Google Sheets
- Airtable
- HubSpot, Salesforce и др.

**Требуемые поля:**
```json
{
  "email": "user@example.com",
  "name": "John Doe",
  "subscribed": true,
  "signup_date": "2026-01-15",
  "last_engagement_days": 5
}
```

## 📊 Структура workflow

```
1. Weekly Schedule (Триггер)
   ↓
2. Get Subscriber List (Получение списка)
   ↓
3. Filter Active Subscribers (Фильтрация)
   ↓
4. Segment Subscribers (Сегментация)
   ↓
5. Route by Segment (Маршрутизация)
   ├─→ New Leads Template → SendGrid
   ├─→ Engaged Users Template → SendGrid
   └─→ Re-engagement Template → SendGrid
      ↓
6. Log Campaign (Airtable) + Track Analytics
   ↓
7. Generate Summary → Send Report
```

## ⚙️ Настройка расписания

По умолчанию workflow запускается **каждый понедельник в 9:00**.

Для изменения расписания отредактируйте ноду "Weekly Schedule":

- **Ежедневно в 10:00**: `0 10 * * *`
- **Каждую среду и пятницу в 14:00**: `0 14 * * 3,5`
- **Первого числа каждого месяца**: `0 9 1 * *`
- **Каждые 3 часа**: `0 */3 * * *`

## 🎨 Персонализация шаблонов

### Изменение дизайна писем

Шаблоны используют встроенный HTML/CSS. Для изменения:

1. Откройте ноду "Send Email (SendGrid)"
2. Найдите параметр "message"
3. Измените HTML-код

### Переменные для персонализации

- `{{$json.personalization.first_name}}` - имя получателя
- `{{$json.personalization.company_name}}` - название компании
- `{{$json.personalization.special_offer}}` - специальное предложение
- `{{$json.personalization.cta_link}}` - ссылка призыва к действию

## 📈 Аналитика и отчетность

### Отслеживание кампаний

Workflow автоматически:
- Логирует отправленные письма в Airtable
- Отправляет данные в систему аналитики
- Генерирует отчеты о завершении кампании

### Отчеты в Slack

После завершения кампании вы получите сообщение:

```
📊 Email Campaign Complete!

Total Sent: 150
New Leads: 45
Engaged: 80
Re-engagement: 25

Timestamp: 2026-02-09T09:15:00Z
```

### Обработка ошибок

При ошибках отправки:
- Автоматическое уведомление в Slack
- Детальная информация об ошибке
- Email получателя для повторной попытки

## 📝 Best Practices

### Доставляемость

1. **Верифицируйте домен** в SendGrid
2. **Настройте SPF, DKIM, DMARC** записи
3. **Прогревайте IP** постепенно увеличивая объемы
4. **Очищайте список** от неактивных адресов

### Контент

1. **Персонализируйте** каждое письмо
2. **Тестируйте темы** писем (A/B тесты)
3. **Оптимизируйте для мобильных** устройств
4. **Добавляйте четкий CTA** (призыв к действию)

### Compliance

1. **Добавьте ссылку отписки** в каждое письмо
2. **Соблюдайте GDPR** и другие регуляции
3. **Получайте явное согласие** на рассылку
4. **Храните данные безопасно**

## 🐛 Troubleshooting

### Письма не отправляются

- Проверьте API ключ SendGrid
- Убедитесь, что email отправителя верифицирован
- Проверьте лимиты SendGrid плана

### Ошибки сегментации

- Убедитесь, что все требуемые поля присутствуют
- Проверьте формат дат (ISO 8601)
- Проверьте логику в ноде "Segment Subscribers"

### Не приходят отчеты

- Проверьте Slack webhook URL
- Убедитесь, что workflow завершается успешно
- Проверьте логи n8n
