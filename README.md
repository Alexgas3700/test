## 🎯 Описание

Этот workflow позволяет автоматизировать процесс создания и публикации контента в VK. Он поддерживает различные типы контента, планирование публикаций, обработку медиа-файлов и интеграцию с внешними системами.

## ✨ Возможности

- **Множественные триггеры**: Запуск по расписанию или через webhook
- **Гибкая настройка контента**: Поддержка текста, изображений, видео и других вложений
- **Планирование публикаций**: Отложенная публикация контента
- **Валидация данных**: Проверка корректности контента перед публикацией
- **Обработка ошибок**: Автоматическое уведомление об ошибках
- **Логирование**: Сохранение истории публикаций (опционально)
- **Геолокация**: Добавление местоположения к постам
- **Хэштеги**: Автоматическое добавление хэштегов
- **Настройки поста**: Управление комментариями, уведомлениями, рекламными метками

## 📦 Требования

### Обязательные

- **n8n** версии 1.0.0 или выше
- **VK API** токен с правами на публикацию постов
- Доступ к группе или странице VK (owner_id)

### Опциональные

- **PostgreSQL** для логирования публикаций
- **Webhook endpoint** для уведомлений о статусе публикаций

## 🚀 Установка

### 1. Установка n8n

```bash
# Через npm
npm install n8n -g

# Через Docker
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n
```

### 2. Импорт workflow

1. Откройте n8n в браузере (обычно http://localhost:5678)
2. Перейдите в раздел "Workflows"
3. Нажмите "Import from File"
4. Выберите файл `vk-content-workflow.json`

## ⚙️ Настройка

### 1. Получение VK API токена

1. Перейдите на https://vk.com/apps?act=manage
2. Создайте новое Standalone-приложение
3. Получите токен доступа с правами:
   - `wall` - для публикации постов
   - `photos` - для загрузки фотографий
   - `video` - для загрузки видео
   - `groups` - для работы с группами

### 2. Настройка credentials в n8n

#### VK OAuth2 API

1. В n8n перейдите в "Credentials" → "New"
2. Выберите "VK OAuth2 API"
3. Заполните поля:
   - **Client ID**: ID вашего приложения
   - **Client Secret**: Защищенный ключ приложения
   - **Access Token**: Токен доступа

#### PostgreSQL (опционально)

1. Создайте credentials для PostgreSQL
2. Заполните данные подключения к БД
3. Создайте таблицу для логирования:

```sql
CREATE TABLE vk_posts (
    id SERIAL PRIMARY KEY,
    post_id VARCHAR(255) NOT NULL,
    message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50),
    owner_id VARCHAR(255),
    attachments TEXT,
    metadata JSONB
);

CREATE INDEX idx_vk_posts_post_id ON vk_posts(post_id);
CREATE INDEX idx_vk_posts_created_at ON vk_posts(created_at);
```

### 3. Настройка owner_id

Получите ID вашей группы или страницы:

```bash
# Для группы (отрицательное значение)
# Если URL: https://vk.com/club123456
# owner_id = -123456

# Для страницы (положительное значение)
# Если URL: https://vk.com/id123456
# owner_id = 123456
```

### 4. Настройка узлов workflow

1. **Schedule Trigger**: Настройте расписание запуска (по умолчанию каждые 2 часа)
2. **Webhook Trigger**: Скопируйте URL webhook для внешних интеграций
3. **Create VK Post**: Укажите credentials VK OAuth2 API
4. **Log to Database**: Укажите credentials PostgreSQL (если используется)
5. **Notify Success/Error**: Настройте webhook для уведомлений (если требуется)

## 📖 Использование

### Запуск через webhook

Отправьте POST-запрос на webhook URL:

```bash
curl -X POST https://your-n8n-instance.com/webhook/create-vk-post \
  -H "Content-Type: application/json" \
  -d '{
    "owner_id": "-123456789",
    "message": "Привет, VK! Это автоматический пост.",
    "attachments": ["photo-123456789_456239017"],
    "hashtags": ["n8n", "automation", "vk"],
    "from_group": 1
  }'
```

### Запуск по расписанию

Workflow автоматически запускается по настроенному расписанию. Для этого необходимо:

1. Подготовить источник данных (например, Google Sheets, Airtable, база данных)
2. Добавить узел для получения данных после Schedule Trigger
3. Передать данные в узел "Check Content"

## 🔧 API и параметры

### Входные параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `owner_id` | string | Да | ID владельца стены (группа/страница) |
| `message` | string | Да | Текст поста |
| `attachments` | array | Нет | Массив вложений (фото, видео, ссылки) |
| `from_group` | number | Нет | Публикация от имени группы (1) или пользователя (0) |
| `publish_date` | string | Нет | Дата и время публикации (ISO 8601) |
| `hashtags` | array | Нет | Массив хэштегов (без #) |
| `lat` | number | Нет | Широта |
| `long` | number | Нет | Долгота |
| `place_id` | number | Нет | ID места |
| `mark_as_ads` | number | Нет | Пометить как рекламу (1/0) |
| `close_comments` | number | Нет | Закрыть комментарии (1/0) |
| `mute_notifications` | number | Нет | Отключить уведомления (1/0) |
| `copyright` | string | Нет | Ссылка на источник |

### Формат вложений

```json
{
  "attachments": [
    "photo-123456789_456239017",
    "video-123456789_456239018",
    "https://example.com/page"
  ]
}
```

### Выходные данные


## 🔒 Безопасность

### Рекомендации

1. **Храните токены безопасно**: Используйте n8n credentials, не храните токены в коде
2. **Ограничьте права токена**: Выдавайте только необходимые разрешения
3. **Используйте HTTPS**: Для webhook endpoints
4. **Валидируйте входные данные**: Проверяйте все данные перед публикацией
5. **Ограничьте доступ к webhook**: Используйте авторизацию (API ключи, JWT)
6. **Мониторинг**: Отслеживайте подозрительную активность

### Настройка авторизации webhook

Добавьте узел "HTTP Request" с проверкой токена перед "Check Content":

```javascript
// В узле Code
const authHeader = $input.item.json.headers['authorization'];
const expectedToken = 'your-secret-token';

if (authHeader !== `Bearer ${expectedToken}`) {
  throw new Error('Unauthorized');
}

return $input.all();
```

## 🔍 Troubleshooting

### Ошибка: "Invalid access token"

**Решение**: 
- Проверьте срок действия токена
- Убедитесь, что токен имеет необходимые права
- Перевыпустите токен в настройках приложения VK

### Ошибка: "Access denied"

**Решение**:
- Проверьте owner_id (для групп должен быть отрицательным)
- Убедитесь, что у приложения есть права на публикацию в группе
- Проверьте, что пользователь является администратором группы

### Ошибка: "Message is too long"

**Решение**:
- Максимальная длина поста в VK - 16384 символа
- Сократите текст или разбейте на несколько постов

### Ошибка: "Invalid attachments"

**Решение**:
- Проверьте формат вложений (должен быть `type{owner_id}_{media_id}`)
- Убедитесь, что медиа-файлы загружены и доступны
- Максимум 10 вложений на пост

### Workflow не запускается по расписанию

**Решение**:
- Активируйте workflow (переключатель в правом верхнем углу)
- Проверьте настройки Schedule Trigger
- Убедитесь, что n8n работает в режиме production

### Не работает webhook

**Решение**:
- Проверьте, что workflow активирован
- Убедитесь, что URL webhook правильный
- Проверьте настройки firewall/proxy
- Убедитесь, что используется правильный HTTP метод (POST)
