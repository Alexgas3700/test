# VK Content Creation Workflow для n8n

## Описание

Этот workflow предназначен для автоматического создания и публикации контента в VKontakte (VK). Workflow включает в себя генерацию контента, валидацию, публикацию и логирование результатов.

## Возможности

- ✅ **Автоматическая публикация контента** по расписанию
- ✅ **Валидация контента** перед публикацией
- ✅ **Обработка ошибок** с детальным логированием
- ✅ **Поддержка текстового контента** и вложений (фото, видео)
- ✅ **Логирование результатов** в базу данных (опционально)
- ✅ **Уведомления** о результатах публикации (опционально)
- ✅ **Сбор статистики** опубликованных постов (опционально)

## Структура Workflow

### 1. Schedule Trigger (Триггер по расписанию)
- Запускает workflow каждые 6 часов
- Можно настроить на любой интервал (часы, дни, недели)

### 2. Generate Content (Генерация контента)
- Генерирует контент для публикации
- Использует шаблоны сообщений
- Можно интегрировать с AI для генерации уникального контента

### 3. Validate Content (Валидация контента)
- Проверяет длину сообщения (максимум 4096 символов)
- Проверяет наличие обязательных полей
- Предотвращает публикацию некорректного контента

### 4. Check Validation (Проверка валидации)
- Направляет workflow в зависимости от результата валидации
- При ошибке - переход к обработке ошибок
- При успехе - переход к публикации

### 5. VK Post Content (Публикация в VK)
- Отправляет пост в VK через API
- Поддерживает текст и вложения
- Использует VK API v5.131

### 6. Check Success (Проверка успешности)
- Проверяет ответ от VK API
- Определяет успешность публикации
- Направляет на логирование

### 7. Log Success / Log Error (Логирование)
- Сохраняет результаты в базу данных (опционально)
- Записывает ID поста, сообщение, статус, время

### 8. Get Post Stats (Получение статистики)
- Получает статистику опубликованного поста (опционально)
- Собирает данные о просмотрах, лайках, репостах

### 9. Send Notification (Отправка уведомлений)
- Отправляет email-уведомление о результате (опционально)
- Можно заменить на Telegram, Slack и т.д.

## Установка и настройка

### Шаг 1: Импорт workflow в n8n

1. Откройте n8n
2. Нажмите "Import from File"
3. Выберите файл `vk-content-workflow.json`
4. Workflow будет импортирован

### Шаг 2: Получение VK Access Token

1. Перейдите на [VK Developers](https://dev.vk.com/)
2. Создайте Standalone приложение
3. Получите access token с правами:
   - `wall` - для публикации на стене
   - `photos` - для загрузки фотографий (если нужно)
   - `groups` - для работы с группами (если публикуете в группу)

**Способ получения токена:**

```
https://oauth.vk.com/authorize?client_id=YOUR_APP_ID&display=page&redirect_uri=https://oauth.vk.com/blank.html&scope=wall,photos,groups&response_type=token&v=5.131
```

Замените `YOUR_APP_ID` на ID вашего приложения.

### Шаг 3: Настройка credentials в n8n

1. В n8n перейдите в "Credentials"
2. Создайте новый credential типа "HTTP Request"
3. Добавьте параметр `vkAccessToken` со значением вашего access token
4. Сохраните credential

### Шаг 4: Настройка параметров workflow

Откройте узел **Generate Content** и измените:

```javascript
const owner_id = "-YOUR_GROUP_ID"; // Замените на ID вашей группы
```

**Как узнать ID группы:**
- Откройте группу в VK
- ID находится в URL: `vk.com/club123456` → ID = `-123456` (с минусом для групп)
- Для личной страницы используйте положительный ID

### Шаг 5: Настройка расписания

В узле **Schedule Trigger** настройте частоту запуска:
- Каждые N часов
- В определенное время
- По дням недели
- И т.д.

## Кастомизация контента

### Использование шаблонов

В узле **Generate Content** можно настроить шаблоны сообщений:

```javascript
const contentTemplates = [
  "🚀 Ваш текст 1",
  "💡 Ваш текст 2",
  "🌟 Ваш текст 3"
];
```

### Интеграция с AI для генерации контента

Добавьте узел **OpenAI** или **Anthropic** перед узлом Generate Content:

1. Добавьте узел OpenAI
2. Настройте промпт для генерации контента
3. Передайте результат в узел Generate Content

Пример промпта:
```
Сгенерируй мотивационный пост для VK на тему саморазвития. 
Длина: 200-300 символов. 
Добавь 2-3 хештега.
```

### Добавление изображений

Для публикации с изображениями:

1. Включите узел **Get Upload Server**
2. Загрузите изображение через полученный URL
3. Сохраните изображение через `photos.saveWallPhoto`
4. Добавьте attachment в формате: `photo{owner_id}_{photo_id}`

Пример:
```javascript
const attachments = "photo-123456_789012";
```

## Настройка логирования

### База данных PostgreSQL

1. Создайте таблицу для логов:

```sql
CREATE TABLE vk_posts_log (
  id SERIAL PRIMARY KEY,
  post_id VARCHAR(255),
  owner_id VARCHAR(255),
  message TEXT,
  status VARCHAR(50),
  error TEXT,
  timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

2. Включите узлы **Log Success** и **Log Error**
3. Настройте подключение к PostgreSQL в credentials

### Альтернативные варианты логирования

- **Google Sheets** - для простого просмотра логов
- **Airtable** - для структурированного хранения
- **MongoDB** - для NoSQL хранения
- **Файл JSON** - для локального логирования

## Обработка ошибок

Workflow включает обработку следующих ошибок:

1. **Ошибки валидации**
   - Пустое сообщение
   - Превышение лимита символов
   - Отсутствие owner_id

2. **Ошибки VK API**
   - Невалидный access token
   - Недостаточно прав
   - Rate limit exceeded
   - Ошибки сети

3. **Обработка**
   - Логирование ошибки
   - Отправка уведомления
   - Повторная попытка (можно добавить)

## Безопасность

### Рекомендации по безопасности

1. **Храните access token в безопасности**
   - Используйте n8n credentials
   - Не храните токен в коде
   - Регулярно обновляйте токен

2. **Ограничьте права токена**
   - Запрашивайте только необходимые права
   - Используйте токен группы вместо личного

3. **Настройте rate limiting**
   - VK API имеет лимиты на количество запросов
   - Добавьте задержки между запросами
   - Обрабатывайте ошибки rate limit

4. **Резервное копирование**
   - Экспортируйте workflow регулярно
   - Сохраняйте версии workflow
   - Делайте backup базы данных с логами

## Troubleshooting

### Проблема: "Invalid access token"

**Решение:**
- Проверьте, что токен действителен
- Убедитесь, что токен имеет необходимые права
- Получите новый токен

### Проблема: "Access denied"

**Решение:**
- Проверьте права токена
- Убедитесь, что вы администратор группы
- Проверьте настройки приватности группы

### Проблема: "Message is too long"

**Решение:**
- Проверьте длину сообщения (максимум 4096 символов)
- Добавьте обрезку текста в Generate Content
- Используйте сокращения

### Проблема: "Rate limit exceeded"

**Решение:**
- Уменьшите частоту публикаций
- Добавьте задержки между запросами
- Используйте execute метод для батчинга

## Примеры использования

### Пример 1: Ежедневные мотивационные посты

```javascript
// В Generate Content
const motivationalQuotes = [
  "Успех - это сумма маленьких усилий, повторяемых изо дня в день.",
```

## FAQ

**Q: Можно ли публиковать видео?**
A: Да, используйте метод `video.save` для загрузки видео и добавьте attachment в формате `video{owner_id}_{video_id}`.

**Q: Как публиковать в несколько групп?**
A: Создайте цикл или дублируйте узел VK Post Content с разными owner_id.

**Q: Можно ли планировать посты на будущее?**
A: Да, используйте параметр `publish_date` в методе `wall.post` (Unix timestamp).

**Q: Как удалить пост в случае ошибки?**
A: Используйте метод `wall.delete` с параметрами `owner_id` и `post_id`.

**Q: Можно ли публиковать от имени группы?**
A: Да, используйте параметр `from_group=1` в методе `wall.post`.
