## Предварительные требования

### Необходимое ПО

- ✅ **n8n** версии 1.0 или выше
- ✅ **SMTP сервер** для отправки email
- ✅ **База данных** с контактами предпринимателей
- ✅ **Веб-сервер** для landing page

### Рекомендуемые сервисы

**SMTP провайдеры:**
- SendGrid (99,000 писем/месяц бесплатно)
- Mailgun (5,000 писем/месяц бесплатно)
- Amazon SES (62,000 писем/месяц бесплатно на AWS)
- SMTP.ru (для российского рынка)

**Хостинг n8n:**
- n8n Cloud (официальный облачный сервис)
- Self-hosted на VPS (DigitalOcean, AWS, Azure)
- Docker контейнер

**База данных:**
- PostgreSQL
- MySQL
- MongoDB
- REST API endpoint

## Установка workflow

### Шаг 1: Импорт workflow в n8n

1. Откройте n8n UI в браузере
2. Перейдите в раздел **Workflows**
3. Нажмите кнопку **Import from File**
4. Выберите файл `n8n-customer-acquisition-workflow.json`
5. Нажмите **Import**

### Шаг 2: Проверка импорта

После импорта вы должны увидеть workflow с 9 узлами:

```
Schedule Trigger → Fetch Entrepreneurs → Filter Contacts → 
Prepare Data → Send Email → Log Campaign → Summary
                    ↓ (error)
              Error Handler → Log Error
```

### Шаг 3: Активация workflow

**Важно:** Не активируйте workflow до завершения всех настроек!

## Настройка учетных данных

### SMTP Credentials (для отправки email)

1. В n8n перейдите в **Settings → Credentials**
2. Нажмите **Add Credential**
3. Выберите **SMTP**
4. Заполните данные:

```
Name: SMTP Account
Host: smtp.yourprovider.com
Port: 587 (или 465 для SSL)
SSL/TLS: Enabled
User: your-smtp-username
Password: your-smtp-password
```

**Примеры настроек для популярных провайдеров:**

#### SendGrid
```
Host: smtp.sendgrid.net
Port: 587
User: apikey
Password: <your-sendgrid-api-key>
```

#### Gmail (для тестирования)
```
Host: smtp.gmail.com
Port: 587
User: your-email@gmail.com
Password: <app-specific-password>
```

#### Mailgun
```
Host: smtp.mailgun.org
Port: 587
User: postmaster@your-domain.com
Password: <your-mailgun-smtp-password>
```

### HTTP Basic Auth (для базы данных предпринимателей)

1. В **Credentials** нажмите **Add Credential**
2. Выберите **HTTP Basic Auth**
3. Заполните:

```
Name: Entrepreneurs DB Auth
User: your-api-username
Password: your-api-password
```

### HTTP Header Auth (для CRM API - опционально)

1. В **Credentials** нажмите **Add Credential**
2. Выберите **HTTP Header Auth**
3. Заполните:

```
Name: CRM API Auth
Header Name: Authorization
Value: Bearer <your-api-token>
```

## Настройка переменных окружения

### Шаг 1: Добавление переменных в n8n

1. Перейдите в **Settings → Variables**
2. Нажмите **Add Variable** для каждой переменной

### Обязательные переменные

```bash
# База данных предпринимателей
ENTREPRENEURS_DB_URL=https://your-database-api.com

# Email отправителя
SENDER_EMAIL=marketing@yourcompany.com

# Landing page для предложения
LANDING_PAGE_URL=https://yourcompany.com/claim-offer

# Страница отписки
UNSUBSCRIBE_URL=https://yourcompany.com/unsubscribe

# Политика конфиденциальности
PRIVACY_URL=https://yourcompany.com/privacy
```

### Опциональные переменные

```bash
# CRM API для логирования
CRM_API_URL=https://your-crm-api.com

# Сервис логирования ошибок
ERROR_LOG_URL=https://your-logging-service.com

# BCC для аналитики
BCC_EMAIL=analytics@yourcompany.com
```

### Шаг 2: Проверка переменных

В любом узле workflow вы можете проверить переменные:

```javascript
{{ $env.SENDER_EMAIL }}
```

## Настройка базы данных

### Требования к API базы данных

Ваш API должен:

1. **Принимать GET запрос** на endpoint `/api/entrepreneurs`
2. **Поддерживать query параметры** для фильтрации
3. **Возвращать JSON** с массивом объектов
4. **Требовать аутентификацию** (Basic Auth или API Key)

### Формат ответа API

```json
[
  {
    "id": "ENT-10001",
    "email": "entrepreneur@example.com",
    "first_name": "Иван",
    "last_name": "Петров",
    "company_name": "ТехСтарт",
    "business_type": "entrepreneur",
    "industry": "tech",
    "company_size": 5,
    "region": "moscow",
    "consent_marketing": true,
    "email_verified": true,
    "registration_date": "2026-01-15T10:30:00Z"
  }
]
```

### Обязательные поля

- `id` - Уникальный идентификатор
- `email` - Email адрес
- `first_name` - Имя
- `company_name` - Название компании
- `business_type` - Должен быть "entrepreneur"
- `industry` - Отрасль (для расчета скидки)
- `consent_marketing` - Согласие на маркетинг (true)
- `email_verified` - Подтвержденный email (true)

### Пример API endpoint

```bash
GET https://your-api.com/api/entrepreneurs?status=potential&segment=entrepreneur&email_verified=true

Headers:
Authorization: Basic <base64-encoded-credentials>
```

### Если у вас нет API

Если у вас пока нет API, вы можете:

1. **Использовать Google Sheets + n8n Google Sheets node**
2. **Создать простой API с Airtable**
3. **Использовать Webhook node с статическими данными**
4. **Настроить базу данных с Supabase (бесплатно)**

#### Пример с Google Sheets:

1. Создайте Google Sheet с колонками из примера
2. В workflow замените "Fetch Potential Entrepreneurs" на "Google Sheets" node
3. Настройте чтение данных из таблицы

2. **НЕ активируйте** workflow (чтобы не запускался по расписанию)

4. Проверьте каждый узел на наличие ошибок
5. Убедитесь, что email пришел на тестовый адрес

- ✅ Правильное имя получателя

- ✅ Работают все ссылки
- ✅ Корректное отображение на мобильных устройствах
- ✅ Не попал в спам

### Шаг 4: Тестирование на разных email клиентах

Отправьте тестовые письма на:

- Gmail

Убедитесь, что:

## Запуск в production

### Шаг 1: Финальная проверка

Перед активацией убедитесь:

- ✅ Все credentials настроены
- ✅ Все переменные окружения заданы
- ✅ Тестовая отправка прошла успешно
- ✅ База данных возвращает корректные данные
- ✅ Landing page готова принимать промокоды
- ✅ Система отписки работает
- ✅ SMTP лимиты достаточны для вашего объема

### Шаг 2: Настройка расписания

По умолчанию workflow запускается каждый понедельник в 9:00 (Europe/Moscow).

Для изменения:

1. Откройте узел **Schedule Trigger**
2. Измените cron expression:

```
0 9 * * 1  # Каждый понедельник в 9:00
0 9 * * *  # Каждый день в 9:00
0 14 * * 5 # Каждую пятницу в 14:00
0 9 1 * *  # Первое число месяца в 9:00
```

3. Установите правильный часовой пояс в **Workflow Settings**

### Шаг 3: Активация workflow

1. В правом верхнем углу переключите **Inactive → Active**
2. Workflow теперь будет запускаться автоматически по расписанию

### Шаг 4: Первый запуск

Следите за первым автоматическим запуском:

1. Проверьте **Executions** в n8n
2. Убедитесь, что все письма отправлены
3. Проверьте логи на ошибки
4. Мониторьте bounce rate и spam complaints

## Мониторинг и аналитика

### Метрики для отслеживания

1. **Отправка:**
   - Количество отправленных писем
   - Процент успешных отправок
   - Количество ошибок

2. **Доставка:**
   - Delivery rate (процент доставленных)
   - Bounce rate (процент отклоненных)
   - Spam complaints

3. **Вовлеченность:**
   - Open rate (процент открытий)
   - Click-through rate (процент кликов)
   - Conversion rate (процент использования промокодов)

4. **Бизнес-метрики:**
   - Количество активированных промокодов
   - Выручка от кампании
   - ROI (возврат инвестиций)
   - Customer Acquisition Cost (CAC)

### Настройка аналитики

#### В n8n

1. Проверяйте **Executions** после каждого запуска
2. Используйте **Campaign Summary** node для сводки
3. Настройте уведомления об ошибках

#### В SMTP провайдере

1. Настройте webhook для событий (delivered, opened, clicked)
2. Отслеживайте bounce rate
3. Мониторьте spam complaints

#### В CRM

1. Отслеживайте использование промокодов
2. Связывайте продажи с email кампаниями
3. Рассчитывайте ROI

### Оптимизация кампании

На основе метрик оптимизируйте:

1. **Низкий Open Rate (<20%):**
   - Улучшите subject line
   - Измените время отправки
   - Проверьте sender reputation

2. **Низкий Click Rate (<3%):**
   - Улучшите CTA
   - Сделайте предложение более привлекательным
   - Упростите дизайн письма

3. **Низкая конверсия (<1%):**
   - Увеличьте размер скидки
   - Упростите процесс активации
   - Добавьте urgency (срочность)

4. **Высокий Bounce Rate (>5%):**
   - Очистите базу от невалидных email
   - Используйте email validation сервис
   - Проверьте SPF/DKIM настройки

### Регулярное обслуживание

**Еженедельно:**
- Проверяйте executions на ошибки
- Анализируйте метрики кампании
- Обновляйте базу данных

**Ежемесячно:**
- Очищайте базу от отписавшихся
- Анализируйте ROI
- A/B тестируйте новые варианты
- Обновляйте email шаблон

**Ежеквартально:**
- Пересматривайте стратегию скидок
- Обновляйте сегментацию аудитории
- Оптимизируйте workflow
- Проверяйте соответствие GDPR

## Устранение проблем

### Workflow не запускается по расписанию

**Проблема:** Workflow активен, но не выполняется автоматически

**Решение:**
1. Проверьте, что workflow **Active**
2. Проверьте cron expression в Schedule Trigger
3. Убедитесь, что n8n instance запущен
4. Проверьте часовой пояс в настройках workflow
5. Проверьте логи n8n на ошибки

### Письма не отправляются

**Проблема:** Execution завершается с ошибкой на узле Send Email

**Решение:**
1. Проверьте SMTP credentials
2. Убедитесь, что SMTP порт открыт
3. Проверьте лимиты SMTP провайдера
4. Проверьте, что sender email подтвержден
5. Попробуйте отправить тестовое письмо через SMTP клиент

### База данных не возвращает данные

**Проблема:** Fetch Entrepreneurs node возвращает пустой массив

**Решение:**
1. Проверьте URL API
2. Проверьте authentication credentials
3. Проверьте query параметры
4. Протестируйте API через Postman/curl
5. Проверьте, есть ли данные в базе, соответствующие фильтрам

### Промокоды не уникальны

**Проблема:** Разные клиенты получают одинаковые промокоды

**Решение:**
1. Убедитесь, что ID клиента уникален
2. Добавьте timestamp в промокод
3. Используйте UUID вместо ID
4. Проверяйте уникальность перед отправкой

### Письма попадают в спам

**Проблема:** Высокий процент писем в папке спам

**Решение:**
1. Настройте SPF запись для домена
2. Настройте DKIM подпись
3. Настройте DMARC политику
4. Используйте подтвержденный sender domain
5. Избегайте спам-слов в subject и теле
6. Добавьте физический адрес в footer
7. Используйте double opt-in для подписки

## Дополнительные возможности

### A/B тестирование

Создайте два варианта workflow для тестирования:

1. **Вариант A:** Скидка 20%, стандартный дизайн
2. **Вариант B:** Скидка 25%, новый дизайн

Разделите аудиторию 50/50 и сравните конверсию.

Создайте отдельные workflows для разных сегментов:

1. **Tech предприниматели** - 25% скидка, технические преимущества
2. **Retail предприниматели** - 20% скидка, преимущества для торговли
3. **Services предприниматели** - 20% скидка, преимущества для услуг

Создайте дополнительный workflow для напоминаний:

1. Через 7 дней после первого письма
