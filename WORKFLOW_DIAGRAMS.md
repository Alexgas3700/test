# Диаграммы Workflow

Визуальное представление всех workflow для лучшего понимания структуры.

## 1. Базовый Workflow (vk-post-workflow.json)

### Схема потока данных

```
┌─────────────────────┐
│  Manual Trigger     │  Запуск вручную
│  (Кнопка Test)      │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Set Post Data      │  Настройка данных:
│                     │  - postText: "текст"
│                     │  - imageUrl: "url"
│                     │  - groupId: "123"
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Download Image     │  HTTP Request
│                     │  GET imageUrl
│                     │  → Binary data
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Upload Photo       │  VK API
│  to VK              │  photos.upload
│                     │  → photo ID
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Create VK Post     │  VK API
│                     │  wall.post
│                     │  → post ID
└─────────────────────┘
```

### Данные на каждом этапе

```
Этап 1: Manual Trigger
Output: {}

Этап 2: Set Post Data
Output: {
  postText: "Hello from n8n!",
  imageUrl: "https://example.com/image.jpg",
  groupId: "123456789"
}

Этап 3: Download Image
Output: {
  postText: "Hello from n8n!",
  imageUrl: "https://example.com/image.jpg",
  groupId: "123456789",
  binary: {
    data: <Buffer...>,
    mimeType: "image/jpeg",
    fileName: "image.jpg"
  }
}

Этап 4: Upload Photo to VK
Output: {
  id: "photo-123456789_456123",
  album_id: -7,
  owner_id: -123456789,
  sizes: [...],
  postText: "Hello from n8n!",
  groupId: "123456789"
}

Этап 5: Create VK Post
Output: {
  post_id: 123,
  success: 1
}
```

## 2. Workflow с несколькими изображениями (vk-post-multiple-images-workflow.json)

### Схема потока данных

```
┌─────────────────────┐
│  Manual Trigger     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Set Post Data      │  imageUrls: [url1, url2, url3]
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Prepare for Split  │  Копирование данных
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Split Images       │  Разделение массива
│                     │  3 элемента → 3 итерации
└──────────┬──────────┘
           │
           ▼
    ┌──────────────────────────┐
    │  LOOP (для каждого URL)  │
    │  ┌─────────────────────┐ │
    │  │  Download Image     │ │  Загрузка изображения
    │  └──────────┬──────────┘ │
    │             ▼             │
    │  ┌─────────────────────┐ │
    │  │  Upload Photo to VK │ │  Загрузка на VK
    │  └──────────┬──────────┘ │
    └─────────────┼─────────────┘
                  │
                  ▼
┌─────────────────────┐
│  Aggregate Photos   │  Сбор всех результатов
│                     │  [photo1, photo2, photo3]
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Format Attachments │  Форматирование:
│                     │  "photo1,photo2,photo3"
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Create VK Post     │  Пост с 3 фото
└─────────────────────┘
```

### Пример данных

```
Input (Set Post Data):
{
  postText: "Check out these photos!",
  imageUrls: [
    "https://example.com/photo1.jpg",
    "https://example.com/photo2.jpg",
    "https://example.com/photo3.jpg"
  ],
  groupId: "123456789"
}

After Split (3 iterations):
Iteration 1: { imageUrl: "https://example.com/photo1.jpg", ... }
Iteration 2: { imageUrl: "https://example.com/photo2.jpg", ... }
Iteration 3: { imageUrl: "https://example.com/photo3.jpg", ... }

After Upload (3 results):
[
  { id: "photo-123_1", ... },
  { id: "photo-123_2", ... },
  { id: "photo-123_3", ... }
]

After Format:
{
  attachments: "photo-123_1,photo-123_2,photo-123_3",
  postText: "Check out these photos!",
  groupId: "123456789"
}
```

## 3. Автоматизированный Workflow (vk-scheduled-post-workflow.json)

### Схема потока данных с обработкой ошибок

```
┌─────────────────────┐
│  Schedule Trigger   │  Cron: 0 9 * * *
│  (Каждый день 9:00) │  (Каждый день в 9:00)
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Get Post Content   │  HTTP Request к API
│  from API           │  → получение контента
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Validate Content   │  IF: postText && imageUrl
└──────────┬──────────┘
           │
      ┌────┴────┐
      │         │
    TRUE      FALSE
      │         │
      ▼         ▼
┌─────────┐  ┌──────────────────┐
│ SUCCESS │  │ ERROR PATH       │
│ PATH    │  │                  │
└────┬────┘  └────┬─────────────┘
     │            │
     │            ▼
     │       ┌─────────────────────┐
     │       │  Set Error Response │
     │       └──────────┬──────────┘
     │                  │
     │                  ▼
     │       ┌─────────────────────┐
     │       │  Send Error         │
     │       │  Notification       │
     │       │  (Telegram)         │
     │       └─────────────────────┘
     │
     ▼
┌─────────────────────┐
│  Download Image     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Upload Photo to VK │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Create VK Post     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Set Success        │
│  Response           │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  Send Success       │
│  Notification       │
│  (Telegram)         │
└─────────────────────┘
```

### Логика обработки ошибок

```
┌──────────────────────────────────────────┐
│  Validate Content                        │
│                                          │
│  Условия:                                │
│  1. postText не пустой                   │
│  2. imageUrl не пустой                   │
│                                          │
│  IF (условия выполнены):                 │
│    → SUCCESS PATH (публикация)           │
│  ELSE:                                   │
│    → ERROR PATH (уведомление об ошибке)  │
└──────────────────────────────────────────┘

SUCCESS PATH:
  1. Download Image
  2. Upload to VK
  3. Create Post
  4. Set Success Response
  5. Send Telegram: "✅ Post Published!"

ERROR PATH:
  1. Set Error Response
  2. Send Telegram: "❌ Validation Failed!"
```

## Оптимизация производительности

### Последовательная vs Параллельная обработка

#### Последовательная (медленнее)

```
Image 1: Download → Upload → [3 sec]
Image 2: Download → Upload → [3 sec]
Image 3: Download → Upload → [3 sec]
─────────────────────────────────────
Total: 9 seconds
```

#### Параллельная (быстрее)

```
Image 1: Download → Upload ┐
Image 2: Download → Upload ├→ [3 sec]
Image 3: Download → Upload ┘
─────────────────────────────
Total: 3 seconds
```

### Реализация в n8n

```
┌─────────────────────┐
│  Split In Batches   │  batchSize: 3
└──────────┬──────────┘
           │
     ┌─────┼─────┐
     │     │     │
     ▼     ▼     ▼
   [1]   [2]   [3]  ← Параллельная обработка
     │     │     │
     └─────┼─────┘
           │
           ▼
┌─────────────────────┐
│  Aggregate          │
└─────────────────────┘
```

## Заключение

Эти диаграммы помогут вам:

1. **Понять структуру** каждого workflow
2. **Выбрать подходящий** workflow для вашей задачи
3. **Расширить функциональность** добавлением новых узлов
4. **Оптимизировать производительность** правильной организацией
5. **Отладить проблемы** понимая поток данных
