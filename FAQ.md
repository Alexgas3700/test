## Общие вопросы

### Что такое n8n?

n8n - это инструмент для автоматизации workflow с открытым исходным кодом. Он позволяет соединять различные сервисы и автоматизировать задачи без написания кода.

**Официальный сайт:** https://n8n.io

### Нужно ли мне знать программирование?

**Нет**, базовые функции workflow работают без программирования. Однако знание JavaScript будет полезно для:
- Кастомизации шаблонов писем
- Создания сложной логики
- Расширенной персонализации

### Сколько стоит использование?

**n8n:**
- Self-hosted: бесплатно (open-source)
- n8n Cloud: от $20/месяц

**SMTP сервисы:**
- Gmail: бесплатно (до 500 писем/день)
- SendGrid: бесплатно (до 100 писем/день)
- Mailgun: бесплатно (до 5,000 писем/месяц)

**Google Sheets:** бесплатно

### Можно ли использовать для коммерческих целей?

**Да**, проект распространяется под MIT лицензией, которая разрешает коммерческое использование.

## Установка и настройка

### Какой способ установки n8n выбрать?

| Способ | Плюсы | Минусы | Рекомендуется для |
|--------|-------|--------|-------------------|
| **npm** | Простая установка, легкое обновление | Требует Node.js | Разработчиков |
| **Docker** | Изолированное окружение, легкое развертывание | Требует Docker | Production |
| **n8n Cloud** | Не требует настройки, всегда доступен | Платная подписка | Бизнеса |

**Рекомендация:**
- Для тестирования: npm
- Для production: Docker или n8n Cloud

### Не могу подключиться к Google Sheets

**Проверьте:**

1. **Google Sheets API включен?**
   - Откройте [Google Cloud Console](https://console.cloud.google.com/)
   - APIs & Services → Library
   - Найдите "Google Sheets API" → Enable

2. **OAuth credentials настроены?**
   - Redirect URI должен быть: `http://localhost:5678/rest/oauth2-credential/callback`
   - Для n8n Cloud: `https://ваш-домен.app.n8n.cloud/rest/oauth2-credential/callback`

3. **Авторизация прошла успешно?**
   - В n8n: Credentials → Google Sheets → Reconnect

4. **Есть доступ к таблице?**
   - Откройте таблицу в браузере
   - Проверьте, что вы можете её редактировать

**Быстрое решение для тестирования:**
Сделайте таблицу публичной: Share → Anyone with the link can view

### Gmail не отправляет письма

**Частые причины:**

1. **Используете обычный пароль вместо App Password**
   - ✅ Решение: создайте [App Password](https://myaccount.google.com/apppasswords)
   - Требуется включить 2FA

2. **Неверные SMTP настройки**
   ```
   Host: smtp.gmail.com
   Port: 587
   SSL/TLS: включено
   User: ваш-email@gmail.com
   Password: 16-символьный App Password
   ```

3. **Превышен лимит отправки**
   - Gmail: 500 писем/день
   - Решение: используйте SendGrid или Mailgun

4. **"Less secure app access" заблокирован**
   - Это устаревшая настройка
   - Используйте App Password вместо этого

### Ошибка "Invalid credentials" в n8n

**Решение:**

1. Удалите старый credential:
   - Settings → Credentials → найдите → Delete

2. Создайте новый credential:
   - Add Credential → выберите тип
   - Заполните все поля
   - Нажмите "Connect" или "Save"

3. Обновите узлы workflow:
   - Откройте каждый узел
   - Выберите новый credential
   - Сохраните

## Работа с workflow

### Как импортировать workflow?

1. Скачайте `email-campaign-workflow.json`
2. В n8n: Menu (☰) → Import from File
3. Выберите файл → Import
4. Настройте credentials в узлах

### Workflow не запускается

**Проверьте:**

1. **Workflow активирован?**
   - Переключатель в правом верхнем углу должен быть ON

2. **Триггер настроен?**
   - Manual Trigger: нажмите "Execute Workflow"
   - Schedule Trigger: проверьте расписание

3. **Credentials настроены?**
   - Откройте каждый узел
   - Проверьте, что credential выбран

4. **Есть данные для обработки?**
   - Проверьте Google Sheets
   - Должен быть хотя бы 1 контакт со статусом "Active"

### Как изменить текст письма?

1. Откройте workflow в n8n
2. Найдите узел "Personalize Emails"
3. Двойной клик для открытия
4. Измените переменную `body` в коде
5. Нажмите "Save"

**Быстрый способ:**
Скопируйте готовый шаблон из `email-templates.md`

### Как добавить свой логотип?

В узле "Personalize Emails" добавьте в HTML:

```javascript
const body = `
  <html>
    <body>
      <img src="https://ваш-сайт.com/logo.png" 
           alt="Logo" 
           style="max-width: 200px; height: auto;">
      
      <!-- Остальной контент -->
    </body>
  </html>
`;
```

**Важно:**
- Используйте прямую ссылку на изображение
- Оптимизируйте размер (< 100KB)
- Добавьте alt текст

### Как настроить автоматический запуск?

1. **Удалите Manual Trigger:**
   - Выберите узел → Delete

2. **Добавьте Schedule Trigger:**
   - Нажмите "+" → Trigger → Schedule Trigger
   - Настройте расписание

3. **Примеры расписания:**

```
Каждый день в 9:00:
Rule: 0 9 * * *

Каждый понедельник в 10:00:
Rule: 0 10 * * 1

Каждую пятницу в 15:00:
Rule: 0 15 * * 5

Первое число месяца в 12:00:
Rule: 0 12 1 * *
```

4. **Активируйте workflow:**
   - Переключатель в правом верхнем углу → ON

## Email-рассылки

### Письма попадают в спам

**Причины и решения:**

1. **Отсутствуют SPF/DKIM/DMARC записи**
   - Настройте DNS записи для вашего домена
   - Используйте профессиональный SMTP сервис

2. **Спам-слова в теме или тексте**
   - Избегайте: "БЕСПЛАТНО", "ЗАРАБОТОК", "КРЕДИТ"
   - Не используйте CAPS LOCK
   - Уберите множественные !!!

3. **Низкая репутация отправителя**
   - Начните с малых объемов
   - Постепенно увеличивайте количество
   - Отслеживайте bounce rate

4. **Нет ссылки отписки**
   - Всегда добавляйте unsubscribe ссылку
   - Это требование законодательства

5. **Используете бесплатный email**
   - Вместо @gmail.com используйте @ваш-домен.com
   - Настройте корпоративную почту

**Инструменты проверки:**
- [Mail Tester](https://www.mail-tester.com/) - проверка спам-скора
- [MXToolbox](https://mxtoolbox.com/) - проверка DNS
- [Google Postmaster Tools](https://postmaster.google.com/) - репутация

### Как отследить открытия писем?

**Добавьте tracking pixel:**

```javascript
// В узле "Personalize Emails"
const trackingPixel = `
  <img src="https://ваш-сервер.com/track/open?email=${encodeURIComponent(email)}&campaign=123" 
       width="1" height="1" style="display:none;" />
`;

const body = `
  <html>
    <body>
      ${content}
      ${trackingPixel}
    </body>
  </html>
`;
```

**Примечание:** Требуется настроить сервер для обработки запросов.

**Альтернатива:** Используйте профессиональные ESP (SendGrid, Mailgun), которые предоставляют встроенную аналитику.

### Как отследить клики по ссылкам?

**Добавьте UTM метки:**

```javascript
const url = new URL('https://ваш-сайт.com/offer');
url.searchParams.append('utm_source', 'email');
url.searchParams.append('utm_medium', 'campaign');
url.searchParams.append('utm_campaign', 'winter_2026');
url.searchParams.append('utm_content', email);

const trackedUrl = url.toString();
// https://ваш-сайт.com/offer?utm_source=email&utm_medium=campaign...
```

Затем отслеживайте в Google Analytics.

### Персонализация не работает

**Проверьте:**

1. **Названия колонок в Google Sheets:**
   - Должны быть: `Email`, `Name`, `Company`, `Status`
   - С заглавной буквы
   - Без пробелов

2. **Переменные в коде:**
   ```javascript
   // ✅ Правильно
   const name = item.json.Name;
   
   // ❌ Неправильно
   const name = item.json.name; // маленькая буква
   ```

3. **Данные в таблице:**
   - Проверьте, что ячейки заполнены
   - Нет лишних пробелов

4. **Fallback значения:**
   ```javascript
   const name = item.json.Name || 'Уважаемый клиент';
   ```

### Сколько писем можно отправить за раз?

**Лимиты SMTP провайдеров:**

| Провайдер | Бесплатный план | Платный план |
|-----------|-----------------|--------------|
| Gmail | 500/день | 2,000/день (Google Workspace) |
| SendGrid | 100/день | До 100,000/день |
| Mailgun | 5,000/месяц | До 10,000/день и выше |
| Amazon SES | 200/день (первые 12 месяцев) | Без лимита |

**Рекомендации:**
- Начните с малых объемов (100-200 писем)
- Постепенно увеличивайте
- Используйте батчи с паузами
- Мониторьте bounce rate

## Технические вопросы

### Workflow выполняется слишком долго

**Оптимизация:**

1. **Уменьшите размер батча:**
   ```javascript
   // В узле "Split In Batches"
   batchSize: 5 // вместо 10
   ```

2. **Уменьшите паузу:**
   ```javascript
   // В узле "Wait Between Batches"
   amount: 2 // секунд вместо 5
   ```

3. **Оптимизируйте код:**
   - Уберите ненужные операции
   - Используйте эффективные алгоритмы

4. **Проверьте SMTP сервер:**
   - Медленный сервер = медленная отправка
   - Попробуйте другой провайдер

### Ошибка "Memory limit exceeded"

**Решение:**

1. **Уменьшите количество обрабатываемых контактов:**
   - Разделите список на несколько кампаний

2. **Увеличьте memory limit для n8n:**
   ```bash
   # Docker
   docker run -e NODE_OPTIONS="--max-old-space-size=4096" n8nio/n8n
   
   # npm
   export NODE_OPTIONS="--max-old-space-size=4096"
   n8n start
   ```

3. **Оптимизируйте HTML шаблоны:**
   - Уменьшите размер изображений
   - Минимизируйте CSS

### Как сделать бэкап workflow?

**Экспорт workflow:**

1. В n8n: откройте workflow
2. Menu (☰) → Download
3. Сохраните JSON файл

**Через CLI:**

```bash
n8n export:workflow --id=1 --output=backup.json
```

**Автоматический бэкап:**

Создайте отдельный workflow для бэкапа:
- Schedule Trigger (раз в день)
- HTTP Request к n8n API
- Сохранение в Google Drive / Dropbox

### Можно ли запустить несколько workflow одновременно?

**Да**, n8n поддерживает параллельное выполнение.

**Настройка:**

```bash
# В .env файле или environment variables
N8N_CONCURRENCY_PRODUCTION_LIMIT=10
```

Это позволит запускать до 10 workflow одновременно.

## Безопасность и compliance

### Как соблюдать GDPR?

**Обязательные требования:**

1. **Получите согласие:**
   - Добавьте колонку "Consent Date" в Google Sheets
   - Храните дату и источник согласия

2. **Добавьте в workflow проверку:**
   ```javascript
   if (!item.json.ConsentDate) {
     return null; // Не отправляем без согласия
   }
   ```

3. **Предоставьте возможность отписки:**
   - Ссылка в каждом письме
   - Обработка в течение 30 дней

4. **Право на удаление данных:**
   - Создайте процесс для удаления
   - Удаление в течение 30 дней после запроса

### Безопасно ли хранить credentials в n8n?

**Да**, n8n шифрует credentials.

**Дополнительная безопасность:**

1. **Используйте environment variables:**
   ```bash
   export SMTP_PASSWORD="ваш-пароль"
   ```

2. **Ограничьте доступ к n8n:**
   ```bash
   export N8N_BASIC_AUTH_ACTIVE=true
   export N8N_BASIC_AUTH_USER=admin
   export N8N_BASIC_AUTH_PASSWORD=secure_password
   ```

3. **Используйте HTTPS:**
   - Для production обязательно
   - Настройте SSL сертификат

4. **Регулярно меняйте пароли:**
   - SMTP passwords
   - API keys
   - n8n admin password

### Нужно ли регистрировать обработку данных?

**Зависит от юрисдикции:**

**Россия (152-ФЗ):**
- Да, если обрабатываете персональные данные
- Уведомление в Роскомнадзор

**ЕС (GDPR):**
- Зависит от объема и типа данных
- Консультируйтесь с юристом

**США (CAN-SPAM):**
- Регистрация не требуется
- Соблюдайте требования закона

## Оптимизация и масштабирование

### Как отправить 10,000+ писем?

**Рекомендации:**

1. **Используйте профессиональный ESP:**
   - SendGrid (до 100,000/день)
   - Mailgun (до 10,000/день)
   - Amazon SES (без лимита)

2. **Разделите на партии:**
   ```javascript
   batchSize: 100
   pauseBetweenBatches: 10 // секунд
   ```

3. **Распределите по времени:**
   - Не отправляйте все сразу
   - Растяните на несколько часов/дней

4. **Мониторьте метрики:**
   - Bounce rate должен быть < 5%
   - Complaint rate < 0.1%

5. **Прогрейте IP адрес:**
   - Начните с 100 писем/день
   - Постепенно увеличивайте

### Как улучшить Open Rate?

**Проверенные методы:**

1. **Оптимизируйте тему:**
   - Персонализация: +26% Open Rate
   - Длина 30-50 символов
   - A/B тестирование

2. **Правильное время отправки:**
   - Вторник-четверг
   - 10:00-11:00 по местному времени

3. **Сегментация:**
   - Релевантный контент для каждого сегмента
   - +14% Open Rate

4. **Качество списка:**
   - Удаляйте неактивных подписчиков
   - Регулярно чистите список

5. **От кого письмо:**
   - Используйте имя человека, не компании
   - "Иван из Компании" > "Компания"

### Как масштабировать на несколько доменов?

**Стратегия:**

1. **Создайте отдельные SMTP credentials:**
   - Для каждого домена свой credential

2. **Дублируйте workflow:**
   - Один workflow на домен
   - Или используйте Switch узел для выбора credential

3. **Распределите нагрузку:**
   - Разные домены для разных сегментов
   - Балансировка нагрузки

4. **Мониторинг:**
   - Отслеживайте репутацию каждого домена
   - Используйте Google Postmaster Tools

### Как добавить вложения?

```javascript
{
  "attachments": [

      "filename": "document.pdf",
      "content": "base64_encoded_content",
```

1. Добавить колонку "Language" в Google Sheets
2. Создать шаблоны для каждого языка
3. Выбирать шаблон на основе языка:

```javascript
const templates = {
  'en': englishTemplate,
  'ru': russianTemplate,
  'es': spanishTemplate
};

const language = item.json.Language || 'en';
const template = templates[language];
```
