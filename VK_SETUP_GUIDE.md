## Создание VK приложения

### Шаг 1: Регистрация в VK Developers

1. Перейдите на [VK Developers](https://dev.vk.com/)
2. Войдите в свой аккаунт VK
3. Если это первый раз, примите условия использования

### Шаг 2: Создание нового приложения

1. Нажмите кнопку **"Создать приложение"** или **"Мои приложения"** → **"Создать приложение"**
2. Заполните форму:
   - **Название:** `n8n VK Integration` (или любое другое)
   - **Платформа:** Выберите **"Standalone приложение"**
   - **Категория:** Выберите подходящую категорию (например, "Инструменты")
3. Нажмите **"Создать приложение"**
4. Подтвердите создание через SMS (если требуется)

### Шаг 3: Сохранение учетных данных

После создания приложения вы увидите страницу с настройками. Сохраните:

- **ID приложения (App ID)** - например: `12345678`
- **Защищенный ключ (Secure Key)** - например: `aBcDeFgHiJkLmNoPqRsTuVwXyZ`
- **Сервисный ключ доступа (Service Access Token)** - для серверных запросов

**⚠️ Важно:** Никогда не публикуйте эти данные в открытом доступе!

## Настройка OAuth2

### Шаг 1: Настройка Redirect URI

1. В настройках приложения найдите раздел **"Настройки"**
2. Найдите поле **"Доверенный redirect URI для OAuth"**
3. Добавьте URL вашего n8n сервера:
   ```
   https://your-n8n-domain.com/rest/oauth2-credential/callback
   ```
   Замените `your-n8n-domain.com` на ваш реальный домен

4. Нажмите **"Сохранить"**

### Шаг 2: Настройка базового домена

1. В разделе **"Настройки"** найдите **"Базовый домен"**
2. Укажите домен вашего n8n сервера (без https://):
   ```
   your-n8n-domain.com
   ```
3. Сохраните изменения

## Получение токена доступа

### Метод 1: OAuth2 через n8n (Рекомендуется)

Это самый простой и безопасный способ.

#### В n8n:

1. Откройте n8n
2. Перейдите в **Credentials** (иконка ключа в левом меню)
3. Нажмите **"+ New"** → **"VK"** → **"OAuth2 API"**
4. Заполните поля:
   - **Credential Name:** `VK OAuth2 account`
   - **Client ID:** Ваш App ID
   - **Client Secret:** Ваш Secure Key
   - **Scope:** `photos,wall,groups,offline`
5. Нажмите **"Connect my account"**
6. В открывшемся окне VK авторизуйте приложение
7. После успешной авторизации вернитесь в n8n
8. Нажмите **"Save"**

### Метод 2: Implicit Flow (Ручной способ)

Если OAuth2 через n8n не работает, используйте этот метод.

#### Шаг 1: Формирование URL авторизации

```
https://oauth.vk.com/authorize?client_id=YOUR_APP_ID&display=page&redirect_uri=https://oauth.vk.com/blank.html&scope=photos,wall,groups,offline&response_type=token&v=5.131
```

Замените `YOUR_APP_ID` на ваш реальный App ID.

#### Шаг 2: Получение токена

1. Перейдите по сформированному URL
2. Авторизуйте приложение
3. После авторизации вы будете перенаправлены на `blank.html`
4. В адресной строке браузера найдите параметр `access_token`:
   ```
   https://oauth.vk.com/blank.html#access_token=YOUR_TOKEN&expires_in=0&user_id=123456
   ```
5. Скопируйте значение `YOUR_TOKEN`

#### Шаг 3: Создание credential в n8n

1. В n8n создайте credential типа **"VK API"** (не OAuth2)
2. Вставьте скопированный токен
3. Сохраните

### Метод 3: Сервисный токен (Для серверных приложений)

Используйте сервисный ключ доступа из настроек приложения.

**⚠️ Внимание:** Сервисный токен имеет ограниченные права и не подходит для всех операций.

## Настройка прав доступа

### Основные права (Scope)

При авторизации приложения запросите следующие права:

| Право | Описание | Необходимо для |
|-------|----------|----------------|
| `photos` | Доступ к фотографиям | Загрузка изображений |
| `wall` | Доступ к стене | Публикация постов |
| `groups` | Доступ к группам | Публикация в группах |
| `offline` | Бессрочный токен | Работа без повторной авторизации |

### Полный список прав

Для расширенного функционала можно добавить:

- `video` - работа с видео
- `docs` - работа с документами
- `stories` - работа с историями
- `market` - работа с товарами
- `stats` - доступ к статистике

### Формат Scope

Права указываются через запятую без пробелов:

```
photos,wall,groups,offline
```

## Интеграция с n8n

### Вариант 1: Использование VK OAuth2 credential

**Преимущества:**
- ✅ Автоматическое обновление токена
- ✅ Безопасное хранение
- ✅ Простая настройка

**Настройка в workflow:**

1. Откройте узел VK (например, "Upload Photo to VK")
2. В поле **"Authentication"** выберите `OAuth2`
3. В поле **"Credential to connect with"** выберите созданный credential
4. Сохраните

### Вариант 2: Использование Access Token

**Преимущества:**
- ✅ Работает без OAuth2
- ✅ Подходит для тестирования

**Недостатки:**
- ❌ Токен может истечь
- ❌ Нужно обновлять вручную

**Настройка в workflow:**

1. В узле VK выберите **"Authentication"** → `Access Token`
2. Создайте новый credential типа **"VK API"**
3. Вставьте токен
4. Сохраните

## Настройка для работы с группами

### Получение ID группы

**Способ 1: Из URL**

Если URL группы: `https://vk.com/club123456789`
- ID группы: `123456789`

Если URL группы: `https://vk.com/mygroup`
- Откройте группу
- В адресной строке будет: `https://vk.com/club123456789`
- ID группы: `123456789`

**Способ 2: Через API**

Используйте метод `groups.getById`:

```bash
curl "https://api.vk.com/method/groups.getById?group_id=mygroup&access_token=YOUR_TOKEN&v=5.131"
```

### Настройка прав администратора

Для публикации в группе вам нужны права администратора:

1. Откройте группу VK
2. Перейдите в **"Управление"** → **"Настройки"**
3. Убедитесь, что вы администратор группы
4. В разделе **"Работа с API"** включите нужные функции

### Настройка приложения для группы

1. В настройках группы перейдите в **"Управление"** → **"Приложения"**
2. Найдите ваше приложение
3. Нажмите **"Настроить"**
4. Включите необходимые права:
   - Публикация от имени группы
   - Загрузка фотографий
   - Управление контентом

## Тестирование настройки

### Проверка токена

Используйте метод `users.get` для проверки токена:

```bash
curl "https://api.vk.com/method/users.get?access_token=YOUR_TOKEN&v=5.131"
```

Если токен валиден, вы получите информацию о пользователе.

### Проверка прав

Используйте метод `account.getAppPermissions`:

```bash
curl "https://api.vk.com/method/account.getAppPermissions?access_token=YOUR_TOKEN&v=5.131"
```

Вы получите число, которое является битовой маской прав.

### Тестовая публикация

Создайте простой workflow в n8n:

1. **Manual Trigger**
2. **Set** - установите `groupId` и `postText`
3. **VK** - создайте пост

Запустите workflow и проверьте, что пост появился в группе.

## Решение проблем

### Ошибка: "Invalid access token"

**Причины:**
- Токен истек
- Токен был отозван
- Неправильный формат токена

**Решение:**
1. Получите новый токен
2. Проверьте, что токен скопирован полностью
3. Убедитесь, что используете правильный тип credential

### Ошибка: "Access denied"

**Причины:**
- Недостаточно прав у токена
- Нет прав администратора группы
- Приложение не авторизовано для группы

**Решение:**
1. Проверьте scope при получении токена
2. Убедитесь, что вы администратор группы
3. Авторизуйте приложение для группы

### Ошибка: "Invalid group_id"

**Причины:**
- Неправильный формат ID
- Группа не существует
- Нет доступа к группе

**Решение:**
1. Проверьте формат ID (только цифры)
2. Убедитесь, что группа существует
3. Проверьте права доступа к группе

### Ошибка: "Captcha needed"

**Причины:**
- VK требует подтверждение через капчу
- Слишком много запросов

**Решение:**
1. Уменьшите частоту запросов
2. Используйте официальный OAuth2
3. Подождите некоторое время

### Ошибка: "Too many requests per second"

**Причины:**
- Превышен лимит запросов к API

**Решение:**
1. Добавьте задержку между запросами (узел **Wait**)
2. Используйте батчинг (объединение запросов)
3. Оптимизируйте workflow

## Лимиты VK API

### Лимиты запросов

- **Пользовательский токен:** 3 запроса в секунду
- **Сервисный токен:** 20 запросов в секунду
- **Групповой токен:** 20 запросов в секунду

### Лимиты загрузки

- **Максимальный размер фото:** 50 МБ
- **Максимальное количество фото в посте:** 10
- **Максимальная длина текста поста:** 16384 символа

### Рекомендации

1. Используйте батчинг для множественных операций
2. Кешируйте результаты, где возможно
3. Добавляйте задержки между запросами
4. Обрабатывайте ошибки с retry логикой

## Безопасность

### Лучшие практики

1. **Никогда не публикуйте токены**
   - Не коммитьте в Git
   - Не делитесь в чатах
   - Используйте environment variables

2. **Ограничивайте права**
   - Запрашивайте только необходимые права
   - Используйте минимальный scope

3. **Регулярно обновляйте токены**
   - Используйте `offline` для бессрочных токенов
   - Или настройте автоматическое обновление через OAuth2

4. **Мониторьте использование**
   - Отслеживайте подозрительную активность
   - Проверяйте логи API запросов

5. **Используйте HTTPS**
   - Всегда используйте защищенное соединение
   - Настройте SSL для n8n сервера
